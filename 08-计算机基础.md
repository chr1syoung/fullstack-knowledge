## 8.1 计算机网络

### 8.1.1 HTTP 协议

#### 知识点详解

**HTTP 请求方法：**

| 方法 | 用途 | 幂等 | 安全 |
|------|------|------|------|
| GET | 获取资源 | ✓ | ✓ |
| POST | 创建资源 | ✗ | ✗ |
| PUT | 完整更新资源 | ✓ | ✗ |
| PATCH | 部分更新资源 | ✗ | ✗ |
| DELETE | 删除资源 | ✓ | ✗ |
| HEAD | 获取响应头 | ✓ | ✓ |
| OPTIONS | 获取支持的方法 | ✓ | ✓ |

**HTTP 状态码：**

```
1xx - 信息性
    100 Continue
    101 Switching Protocols

2xx - 成功
    200 OK
    201 Created（创建成功）
    204 No Content（无内容，如 DELETE 成功）
    206 Partial Content（范围请求）

3xx - 重定向
    301 Moved Permanently（永久重定向）
    302 Found（临时重定向）
    304 Not Modified（协商缓存命中）
    307 Temporary Redirect（临时重定向，保持方法）
    308 Permanent Redirect（永久重定向，保持方法）

4xx - 客户端错误
    400 Bad Request（请求格式错误）
    401 Unauthorized（未认证）
    403 Forbidden（无权限）
    404 Not Found（资源不存在）
    405 Method Not Allowed（方法不允许）
    409 Conflict（冲突，如重复创建）
    422 Unprocessable Entity（验证失败）
    429 Too Many Requests（请求过多）

5xx - 服务器错误
    500 Internal Server Error（服务器内部错误）
    502 Bad Gateway（网关错误）
    503 Service Unavailable（服务不可用）
    504 Gateway Timeout（网关超时）
```

**HTTP Headers：**

```
请求头：
    Accept: text/html,application/json
    Accept-Encoding: gzip, deflate, br
    Accept-Language: zh-CN,zh;q=0.9
    Authorization: Bearer <token>
    Cache-Control: no-cache
    Content-Type: application/json
    Cookie: session=xxx
    Host: example.com
    Origin: https://example.com
    Referer: https://example.com/page
    User-Agent: Mozilla/5.0...

响应头：
    Access-Control-Allow-Origin: *
    Cache-Control: max-age=3600
    Content-Encoding: gzip
    Content-Type: application/json; charset=utf-8
    ETag: "abc123"
    Last-Modified: Wed, 21 Oct 2015 07:28:00 GMT
    Location: /new-url（重定向）
    Set-Cookie: session=xxx; HttpOnly; Secure
    X-Content-Type-Options: nosniff
    X-Frame-Options: DENY
```

### 8.1.2 TCP 三次握手与四次挥手

**三次握手（建立连接）：**

```
客户端                          服务器
   |                               |
   |  SYN (seq=x)                  |
   |------------------------------>|
   |                               |
   |  SYN-ACK (seq=y, ack=x+1)    |
   |<------------------------------|
   |                               |
   |  ACK (ack=y+1)                |
   |------------------------------>|
   |                               |
   |  连接建立，开始传输数据         |
```

**为什么需要三次握手：**

1. 确认双方的发送和接收能力都正常
2. 防止已失效的连接请求突然到达服务器
3. 同步双方的初始序列号

**四次挥手（断开连接）：**

```
客户端                          服务器
   |                               |
   |  FIN (seq=x)                  |
   |------------------------------>|  客户端发起关闭
   |                               |
   |  ACK (ack=x+1)                |
   |<------------------------------|  服务器确认
   |                               |
   |  FIN (seq=y)                  |
   |<------------------------------|  服务器发起关闭
   |                               |
   |  ACK (ack=y+1)                |
   |------------------------------>|  客户端确认
   |                               |
   |  等待 2MSL 后关闭              |
```

**为什么需要四次挥手：**

- TCP 是全双工的，双方都需要关闭各自的发送通道
- 服务器收到 FIN 后，可能还有数据要发送，所以 ACK 和 FIN 分开发送

**TIME_WAIT 状态：**

- 客户端发送最后一个 ACK 后，等待 2MSL（最大报文段生存时间）
- 原因：确保服务器收到最后的 ACK；让旧连接的数据包消失

### 8.1.3 HTTPS 原理

**TLS 握手过程：**

```
客户端                          服务器
   |                               |
   |  ClientHello                  |
   |  (支持的加密套件、随机数)       |
   |------------------------------>|
   |                               |
   |  ServerHello                  |
   |  (选择的加密套件、随机数)       |
   |  Certificate（证书）           |
   |  ServerHelloDone              |
   |<------------------------------|
   |                               |
   |  验证证书                      |
   |  ClientKeyExchange            |
   |  (预主密钥，用服务器公钥加密)   |
   |  ChangeCipherSpec             |
   |  Finished                     |
   |------------------------------>|
   |                               |
   |  ChangeCipherSpec             |
   |  Finished                     |
   |<------------------------------|
   |                               |
   |  加密通信开始                  |
```

**密钥生成：**

1. 客户端随机数 + 服务器随机数 + 预主密钥 → 主密钥
2. 主密钥 → 会话密钥（对称加密）

**证书验证：**

1. 检查证书是否由受信任的 CA 签发
2. 检查证书是否过期
3. 检查证书域名是否匹配
4. 检查证书是否被吊销（CRL/OCSP）

### 8.1.4 浏览器缓存

**强缓存：**

```
Cache-Control: max-age=3600    # 缓存 3600 秒
Cache-Control: no-cache        # 不使用强缓存，每次都验证
Cache-Control: no-store        # 不缓存
Cache-Control: public          # 可被代理服务器缓存
Cache-Control: private         # 只能被浏览器缓存
Cache-Control: immutable       # 内容不会改变

Expires: Wed, 21 Oct 2025 07:28:00 GMT  # 过期时间（旧方式）
```

**协商缓存：**

```
# 基于修改时间
Last-Modified: Wed, 21 Oct 2015 07:28:00 GMT  # 响应头
If-Modified-Since: Wed, 21 Oct 2015 07:28:00 GMT  # 请求头

# 基于内容哈希（更精确）
ETag: "abc123"  # 响应头
If-None-Match: "abc123"  # 请求头

# 命中协商缓存：返回 304 Not Modified
```

**缓存策略：**

```
HTML 文件：
    Cache-Control: no-cache（每次验证）

JS/CSS（带 hash）：
    Cache-Control: max-age=31536000, immutable（永久缓存）

图片：
    Cache-Control: max-age=86400（缓存一天）

API 响应：
    Cache-Control: no-store（不缓存）
```

#### 真实面试题

**题目：浏览器缓存机制中的强缓存和协商缓存有什么区别？**

**满分答案：**

**强缓存（不请求服务器）：**
- 命中时直接用本地缓存，返回 `200 (from cache)`
- 控制字段：`Cache-Control: max-age=3600` 或 `Expires`
- `Cache-Control` 优先级高于 `Expires`

**协商缓存（请求服务器验证）：**
- 发请求到服务器，服务器判断是否有更新
- 未更新返回 `304 Not Modified`，有更新返回 `200` + 新内容
- 控制字段：`ETag/If-None-Match`（优先）或 `Last-Modified/If-Modified-Since`

**完整流程：**
```
请求 → 有强缓存且未过期？→ 直接用（200 from cache）
              ↓ 过期
       发请求 + 携带 ETag/Last-Modified
              ↓
       服务器比对 → 未变化 → 304（用本地缓存）
                 → 已变化 → 200（返回新内容）
```

**最佳实践：**
- HTML：`no-cache`（每次协商）
- JS/CSS（带 hash）：`max-age=31536000, immutable`（永久强缓存）

### 8.1.5 CORS 跨域

**同源策略：**

协议 + 域名 + 端口 三者完全相同才是同源。

**CORS 请求类型：**

1. **简单请求**（不触发预检）：
   - 方法：GET、POST、HEAD
   - Content-Type：text/plain、multipart/form-data、application/x-www-form-urlencoded

2. **预检请求**（OPTIONS）：
   - 其他方法（PUT、DELETE 等）
   - 自定义请求头
   - Content-Type: application/json

**服务器配置：**

```javascript
// Express 配置 CORS
const cors = require('cors');

app.use(cors({
    origin: ['https://example.com', 'https://app.example.com'],
    methods: ['GET', 'POST', 'PUT', 'DELETE'],
    allowedHeaders: ['Content-Type', 'Authorization'],
    credentials: true,  // 允许携带 Cookie
    maxAge: 86400       // 预检请求缓存时间
}));

// 手动设置
app.use((req, res, next) => {
    res.header('Access-Control-Allow-Origin', 'https://example.com');
    res.header('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE');
    res.header('Access-Control-Allow-Headers', 'Content-Type, Authorization');
    res.header('Access-Control-Allow-Credentials', 'true');
    
    if (req.method === 'OPTIONS') {
        return res.status(204).send();
    }
    
    next();
});
```

**前端解决跨域的方式：**

```javascript
// 1. 开发环境代理（Vite/Webpack）
```

#### 真实面试题

**题目1：从输入 URL 到页面显示，发生了什么？**

**满分答案：**

**完整流程：**

1. **URL 解析**
   - 解析协议、域名、路径、查询参数
   - 检查 HSTS 列表（是否强制 HTTPS）

2. **DNS 解析**
   - 检查浏览器 DNS 缓存
   - 检查操作系统 DNS 缓存（hosts 文件）
   - 向本地 DNS 服务器查询
   - 递归查询：根域名服务器 → 顶级域名服务器 → 权威域名服务器
   - 返回 IP 地址

3. **建立 TCP 连接**
   - 三次握手
   - 如果是 HTTPS，还需要 TLS 握手

4. **发送 HTTP 请求**
   - 构建请求报文（请求行、请求头、请求体）
   - 发送请求

5. **服务器处理请求**
   - 接收请求
   - 处理业务逻辑
   - 返回响应

6. **接收 HTTP 响应**
   - 检查状态码
   - 处理重定向（3xx）
   - 解压响应体（gzip 等）

7. **浏览器解析渲染**
   - 解析 HTML → 构建 DOM 树
   - 解析 CSS → 构建 CSSOM 树
   - 合并 DOM + CSSOM → 渲染树
   - 布局（Layout）：计算元素位置和大小
   - 绘制（Paint）：绘制像素
   - 合成（Composite）：合并图层

8. **加载子资源**
   - 遇到 CSS、JS、图片等资源，发起新的请求
   - JS 会阻塞 HTML 解析（除非 async/defer）

9. **执行 JavaScript**
   - 解析和执行 JS 代码
   - 可能修改 DOM，触发重排重绘

10. **页面完全加载**
    - DOMContentLoaded 事件（DOM 解析完成）
    - load 事件（所有资源加载完成）

**题目2：HTTP/1.1、HTTP/2、HTTP/3 有什么区别？**

**满分答案：**

**HTTP/1.1：**
- 持久连接（Keep-Alive）
- 管道化（Pipelining，但有队头阻塞问题）
- 每个请求需要完整的头部
- 文本协议

**HTTP/2：**
- **多路复用**：一个 TCP 连接上并发多个请求，解决队头阻塞
- **头部压缩**（HPACK）：减少重复头部传输
- **服务器推送**：服务器主动推送资源
- **二进制协议**：更高效的解析
- **流优先级**：可以设置请求优先级

**HTTP/3：**
- 基于 QUIC 协议（UDP）
- 解决 TCP 层的队头阻塞
- 更快的连接建立（0-RTT）
- 连接迁移（网络切换不断连）
- 内置 TLS 1.3

**对比：**

| 特性 | HTTP/1.1 | HTTP/2 | HTTP/3 |
|------|----------|--------|--------|
| 传输层 | TCP | TCP | QUIC(UDP) |
| 多路复用 | ✗ | ✓ | ✓ |
| 头部压缩 | ✗ | ✓(HPACK) | ✓(QPACK) |
| 服务器推送 | ✗ | ✓ | ✓ |
| 队头阻塞 | 有 | TCP层有 | 无 |
| 连接建立 | 1-RTT | 1-RTT | 0-RTT |

## 8.2 数据结构

### 8.2.1 常见数据结构

**数组与链表：**

```javascript
// 数组：连续内存，随机访问 O(1)，插入删除 O(n)
const arr = [1, 2, 3, 4, 5];

// 链表：非连续内存，随机访问 O(n)，插入删除 O(1)
class ListNode {
    constructor(val, next = null) {
        this.val = val;
        this.next = next;
    }
}

// 反转链表
function reverseList(head) {
    let prev = null;
    let curr = head;
    
    while (curr) {
        const next = curr.next;
        curr.next = prev;
        prev = curr;
        curr = next;
    }
    
    return prev;
}
```

**栈与队列：**

**二叉树：**

```javascript
class TreeNode {
    constructor(val, left = null, right = null) {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}

// 前序遍历（根-左-右）
function preorder(root) {
    if (!root) return [];
    return [root.val, ...preorder(root.left), ...preorder(root.right)];
}

// 中序遍历（左-根-右）
function inorder(root) {
    if (!root) return [];
    return [...inorder(root.left), root.val, ...inorder(root.right)];
}

// 后序遍历（左-右-根）
function postorder(root) {
    if (!root) return [];
    return [...postorder(root.left), ...postorder(root.right), root.val];
}

// 层序遍历（BFS）
function levelOrder(root) {
    if (!root) return [];
    
    const result = [];
    const queue = [root];
    
    while (queue.length) {
        const level = [];
        const size = queue.length;
        
        for (let i = 0; i < size; i++) {
            const node = queue.shift();
            level.push(node.val);
            if (node.left) queue.push(node.left);
            if (node.right) queue.push(node.right);
        }
        
        result.push(level);
    }
    
    return result;
}
```

## 8.3 算法

### 8.3.1 排序算法

```javascript
// 快速排序 O(n log n) 平均
function quickSort(arr) {
    if (arr.length <= 1) return arr;
    
    const pivot = arr[Math.floor(arr.length / 2)];
    const left = arr.filter(x => x < pivot);
    const middle = arr.filter(x => x === pivot);
    const right = arr.filter(x => x > pivot);
    
    return [...quickSort(left), ...middle, ...quickSort(right)];
}

// 归并排序 O(n log n) 稳定
function mergeSort(arr) {
    if (arr.length <= 1) return arr;
    
    const mid = Math.floor(arr.length / 2);
    const left = mergeSort(arr.slice(0, mid));
    const right = mergeSort(arr.slice(mid));
    
    return merge(left, right);
}

function merge(left, right) {
    const result = [];
    let i = 0, j = 0;
    
    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            result.push(left[i++]);
        } else {
            result.push(right[j++]);
        }
    }
    
    return [...result, ...left.slice(i), ...right.slice(j)];
}

// 堆排序 O(n log n)
function heapSort(arr) {
    const n = arr.length;
    
    // 建堆
    for (let i = Math.floor(n / 2) - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }
    
    // 排序
    for (let i = n - 1; i > 0; i--) {
        [arr[0], arr[i]] = [arr[i], arr[0]];
        heapify(arr, i, 0);
    }
    
    return arr;
}

function heapify(arr, n, i) {
    let largest = i;
    const left = 2 * i + 1;
    const right = 2 * i + 2;
    
    if (left < n && arr[left] > arr[largest]) largest = left;
    if (right < n && arr[right] > arr[largest]) largest = right;
    
    if (largest !== i) {
        [arr[i], arr[largest]] = [arr[largest], arr[i]];
        heapify(arr, n, largest);
    }
}
```

**排序算法对比：**

| 算法 | 时间复杂度（平均） | 时间复杂度（最坏） | 空间复杂度 | 稳定性 |
|------|-------------------|-------------------|------------|--------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(1) | 不稳定 |
| 插入排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 快速排序 | O(n log n) | O(n²) | O(log n) | 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n) | 稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | 不稳定 |

### 8.3.2 常见算法题

**防抖（Debounce）：**

```javascript
function debounce(fn, delay, immediate = false) {
    let timer = null;
    
    return function(...args) {
        const context = this;
        
        if (timer) clearTimeout(timer);
        
        if (immediate) {
            const callNow = !timer;
            timer = setTimeout(() => {
                timer = null;
            }, delay);
            if (callNow) fn.apply(context, args);
        } else {
            timer = setTimeout(() => {
                fn.apply(context, args);
                timer = null;
            }, delay);
        }
    };
}

// 使用
const debouncedSearch = debounce((query) => {
    console.log('搜索:', query);
}, 300);
```

**节流（Throttle）：**

```javascript
// 时间戳版本（立即执行）
function throttle(fn, interval) {
    let lastTime = 0;
    
    return function(...args) {
        const now = Date.now();
        
        if (now - lastTime >= interval) {
            fn.apply(this, args);
            lastTime = now;
        }
    };
}

// 定时器版本（延迟执行）
function throttle(fn, interval) {
    let timer = null;
    
    return function(...args) {
        if (!timer) {
            timer = setTimeout(() => {
                fn.apply(this, args);
                timer = null;
            }, interval);
        }
    };
}

// 完整版（首次立即执行，最后一次也执行）
function throttle(fn, interval) {
    let lastTime = 0;
    let timer = null;
    
    return function(...args) {
        const now = Date.now();
        const remaining = interval - (now - lastTime);
        
        if (remaining <= 0) {
            if (timer) {
                clearTimeout(timer);
                timer = null;
            }
            fn.apply(this, args);
            lastTime = now;
        } else if (!timer) {
            timer = setTimeout(() => {
                fn.apply(this, args);
```

**深拷贝：**

```javascript
function deepClone(obj, map = new WeakMap()) {
    // 处理基本类型
    if (obj === null || typeof obj !== 'object') return obj;
    
    // 处理循环引用
    if (map.has(obj)) return map.get(obj);
    
    // 处理特殊类型
    if (obj instanceof Date) return new Date(obj);
    if (obj instanceof RegExp) return new RegExp(obj);
    if (obj instanceof Map) {
        const clone = new Map();
        map.set(obj, clone);
        obj.forEach((value, key) => {
            clone.set(deepClone(key, map), deepClone(value, map));
        });
        return clone;
    }
    if (obj instanceof Set) {
        const clone = new Set();
        map.set(obj, clone);
        obj.forEach(value => {
            clone.add(deepClone(value, map));
        });
        return clone;
    }
    
    // 处理数组和对象
    const clone = Array.isArray(obj) ? [] : {};
    map.set(obj, clone);
    
    for (const key of Reflect.ownKeys(obj)) {
        clone[key] = deepClone(obj[key], map);
    }
    
    return clone;
}
```

```javascript
function binarySearch(arr, target) {
    let left = 0;
    let right = arr.length - 1;
    
    while (left <= right) {
        const mid = Math.floor((left + right) / 2);
        
        if (arr[mid] === target) {
            return mid;
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    
    return -1;
}
```

```javascript
function climbStairs(n) {
    if (n <= 2) return n;

    for (let i = 3; i <= n; i++) {
        [prev, curr] = [curr, prev + curr];
    }
    
    return curr;

// 最长公共子序列
function longestCommonSubsequence(text1, text2) {
    const m = text1.length, n = text2.length;
    const dp = Array.from({ length: m + 1 }, () => new Array(n + 1).fill(0));
    
    for (let i = 1; i <= m; i++) {
        for (let j = 1; j <= n; j++) {
            if (text1[i - 1] === text2[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1] + 1;
            } else {
                dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
            }
        }
    }
    
    return dp[m][n];
}

// 背包问题
function knapsack(weights, values, capacity) {
    const n = weights.length;
    const dp = new Array(capacity + 1).fill(0);
    
    for (let i = 0; i < n; i++) {
        for (let j = capacity; j >= weights[i]; j--) {
            dp[j] = Math.max(dp[j], dp[j - weights[i]] + values[i]);
        }
    }
    
    return dp[capacity];
}
```

**题目：实现一个 LRU 缓存（最近最少使用）**

**满分答案：**

```javascript
class LRUCache {
    constructor(capacity) {
        this.capacity = capacity;
        this.cache = new Map();  // Map 保持插入顺序
    }
    
    get(key) {
        if (!this.cache.has(key)) return -1;
        
        // 将访问的元素移到末尾（最近使用）
        const value = this.cache.get(key);
        this.cache.delete(key);
        this.cache.set(key, value);
        
        return value;
    }
    
    put(key, value) {
        if (this.cache.has(key)) {
            this.cache.delete(key);
        } else if (this.cache.size >= this.capacity) {
            // 删除最久未使用的元素（Map 的第一个元素）
            const firstKey = this.cache.keys().next().value;
            this.cache.delete(firstKey);
        }
        
        this.cache.set(key, value);
    }
}
```

## 8.5 有效括号算法题

#### 知识点详解

**题目：给定一个字符串，判断其是否为有效的括号组合。**

```javascript
// 示例
isValid("()")        // true
isValid("()[]{}")    // true
isValid("(]")        // false
isValid("([)]")      // false
isValid("{[]}")      // true
isValid("")           // true（空字符串）
```

**解题思路：栈（Stack）**

- 遍历字符串
- 遇到左括号 → 入栈
- 遇到右括号 → 检查栈顶是否是对应的左括号
  - 是 → 出栈
  - 否 → 返回 false
- 遍历结束后，栈为空则返回 true

```javascript
function isValid(s) {
    // 括号映射表
    const map = {
        ')': '(',
        ']': '[',
        '}': '{'
    };
    
    // 栈
    const stack = [];
    
    // 遍历字符串
    for (const char of s) {
        // 如果是右括号
        if (map[char]) {
            // 栈顶是否是对应的左括号
            if (stack.pop() !== map[char]) {
                return false;
            }
        } else {
            // 左括号入栈
            stack.push(char);
        }
    }
    
    // 栈为空才是有效括号
    return stack.length === 0;
}
```
