## 15.1 AI 视频编辑器前端架构设计

#### 知识点详解

**核心模块划分：**

```
AI视频编辑器前端架构
├── 1. 媒体管理层（Media Layer）
│   ├── 素材上传（分片/断点续传）
│   ├── 媒体预览（Video/Canvas）
│   └── 资源缓存（IndexedDB）
│
├── 2. 时间轴引擎（Timeline Engine）
│   ├── 轨道管理（视频/音频/字幕/特效）
│   ├── 帧级精度控制
│   └── 拖拽/缩放交互
│
├── 3. AI 能力层（AI Layer）
│   ├── 任务队列（生成/处理任务管理）
│   ├── 流式结果接收（SSE/WebSocket）
│   └── 结果预览与确认
│
├── 4. 渲染引擎（Render Engine）
│   ├── 实时预览（Canvas/WebGL/WebCodecs）
│   ├── 特效合成
│   └── 导出（Web Worker + 编码）
│
├── 5. 状态管理层（State Layer）
│   ├── 编辑历史（Undo/Redo）
│   ├── 协同状态（多人编辑）
│   └── 持久化（自动保存）
│
└── 6. 基础设施层（Infra Layer）
    ├── 性能监控
    ├── 错误边界
    └── 国际化
```

**AI 任务队列：**

```javascript
class AITaskQueue {
    constructor() {
        this.queue = [];
        this.running = new Map();
        this.maxConcurrent = 2;
    }

    async add(task) {
        return new Promise((resolve, reject) => {
            this.queue.push({ ...task, resolve, reject });
            this.flush();
        });
    }

    async flush() {
        while (
            this.running.size < this.maxConcurrent &&
            this.queue.length > 0
        ) {
            const task = this.queue.shift();
            this.running.set(task.id, task);

            this.execute(task)
                .then(task.resolve)
                .catch(task.reject)
                .finally(() => {
                    this.running.delete(task.id);
                    this.flush();
                });
        }
    }

    async execute(task) {
        // 调用 AI 接口，支持流式返回
        const response = await fetch('/api/ai/video', {
            method: 'POST',
            body: JSON.stringify(task.params),
            signal: task.abortController?.signal
        });

        if (task.onProgress) {
            // 流式处理进度
            const reader = response.body.getReader();
            for await (const chunk of readStream(reader)) {
                task.onProgress(chunk);
            }
        }

        return response.json();
    }

    cancel(taskId) {
        const task = this.running.get(taskId);
        task?.abortController?.abort();
    }
}
```

**Undo/Redo 历史栈：**

```javascript
class HistoryManager {
    constructor(maxSize = 50) {
        this.stack = [];
        this.index = -1;
        this.maxSize = maxSize;
    }

    push(command) {
        // 清除 redo 历史
        this.stack = this.stack.slice(0, this.index + 1);
        this.stack.push(command);

        if (this.stack.length > this.maxSize) {
            this.stack.shift();
        } else {
            this.index++;
        }
    }

    undo() {
        if (this.index < 0) return;
        this.stack[this.index].undo();
        this.index--;
    }

    redo() {
        if (this.index >= this.stack.length - 1) return;
        this.index++;
        this.stack[this.index].execute();
    }
}

// Command 模式
class AddClipCommand {

    execute() { this.timeline.addClip(this.clip); }
    undo()    { this.timeline.removeClip(this.clip.id); }
}
```

#### 真实面试题

**题目：如果让你从零设计一个 AI 视频编辑器前端架构，你会考虑哪些核心模块？**

**满分答案：**

分六层设计：

1. **媒体管理层**：分片上传、IndexedDB 缓存、预览生成
2. **时间轴引擎**：多轨道、帧级精度、拖拽交互、Command 模式支持 Undo/Redo

## 15.2 模型生成结果不确定性带来的 UI 挑战

#### 知识点详解

**不确定性的来源：**

```
AI 生成结果不确定性
├── 时间不确定：生成时间 5s ~ 120s 不等
├── 质量不确定：同一 Prompt 每次结果不同
├── 格式不确定：输出可能不符合预期格式
└── 失败不确定：随时可能超时/报错
```

**UI 挑战与解决方案：**

**1. 加载状态设计（时间不确定）**

```jsx
// 骨架屏 + 进度预估
function AIGeneratingState({ estimatedSeconds }) {
    const [elapsed, setElapsed] = useState(0);

    useEffect(() => {
        const timer = setInterval(() => setElapsed(e => e + 1), 1000);
        return () => clearInterval(timer);
    }, []);

    // 非线性进度：前期快，后期慢（避免卡在99%）
    const progress = Math.min(
        (elapsed / estimatedSeconds) * 80 + // 最多到80%
        Math.random() * 5,                   // 随机抖动
        95                                   // 上限95%
    );

    return (
        <div className="generating">
            <SkeletonVideo />
            <ProgressBar value={progress} />
            <p>AI 生成中... 预计还需 {Math.max(0, estimatedSeconds - elapsed)}s</p>
            <button onClick={onCancel}>取消</button>
        </div>
    );
}
```

#### 真实面试题

**题目：在 AI 视频产品中，如何解决模型生成结果不确定性带来的 UI 挑战？**

**满分答案：**

不确定性分四类，对应四种策略：

1. **时间不确定** → 非线性进度条 + 预估剩余时间 + 可取消
2. **质量不确定** → 生成结果先预览，用户确认后才应用；支持重新生成
3. **格式不确定** → 前端做输出校验，不符合格式给出友好提示
4. **失败不确定** → 错误边界 + 重试机制 + 降级到手动操作

**核心原则：** AI 结果不直接替换，给用户控制权；失败要能回滚。

## 15.3 通用 AI Agent 配置化组件设计

**Schema 驱动的组件设计：**

```typescript
interface AgentSchema {
```

**动态表单渲染器：**

```tsx
// 根据 Schema 动态渲染表单
function AgentConfigForm({ schema, values, onChange }) {
    const renderField = (field: InputField) => {
        // 条件显示
        if (field.showWhen) {
            const { field: depField, value: depValue } = field.showWhen;
            if (values[depField] !== depValue) return null;
        }

        switch (field.type) {
            case 'text':
                return (
                    <div key={field.key} className="field">
                        <label>{field.label}</label>
                        <input
                            value={values[field.key] ?? field.default ?? ''}
                            onChange={e => onChange(field.key, e.target.value)}
                            placeholder={field.placeholder}
                        />
                        {field.description && <p className="hint">{field.description}</p>}
                    </div>
                );

            case 'select':
                return (
                    <div key={field.key} className="field">
                        <label>{field.label}</label>
                        <select
                            value={values[field.key] ?? field.default}
                            onChange={e => onChange(field.key, e.target.value)}
                        >
                            {field.options?.map(opt => (
                                <option key={opt.value} value={opt.value}>
                                    {opt.label}
                                </option>
                            ))}
                        </select>
                    </div>
                );

            case 'slider':
                return (
                    <div key={field.key} className="field">
                        <label>{field.label}: {values[field.key] ?? field.default}</label>
                        <input
                            type="range"
                            min={field.min} max={field.max} step={field.step}
                            value={values[field.key] ?? field.default}
                            onChange={e => onChange(field.key, Number(e.target.value))}
                        />
                    </div>
                );

            case 'toggle':
                return (
                    <div key={field.key} className="field field--toggle">
                        <label>{field.label}</label>
                        <input
                            type="checkbox"
                            checked={values[field.key] ?? field.default}
                            onChange={e => onChange(field.key, e.target.checked)}
                        />
                    </div>
                );

            default:
                return null;
        }
    };

    return (
        <form className="agent-config-form">
            {schema.inputs.map(renderField)}
        </form>
    );
}
```

**Prompt 模板填充：**

```typescript
// 将用户配置填入 Prompt 模板
function buildPrompt(schema: AgentSchema, values: Record<string, any>): string {
    let prompt = schema.promptTemplate;

    for (const [key, value] of Object.entries(values)) {
        prompt = prompt.replace(new RegExp(`{{${key}}}`, 'g'), String(value));
    }

    // 清理未填充的可选变量
    prompt = prompt.replace(/{{[^}]+}}/g, '');

    return prompt.trim();
}
```

**完整 Agent 组件：**

```tsx
function AIAgentPanel({ schema }: { schema: AgentSchema }) {
    const [values, setValues] = useState<Record<string, any>>({});
    const [status, setStatus] = useState<'idle' | 'running' | 'done' | 'error'>('idle');
    const [result, setResult] = useState<any>(null);

    const handleChange = (key: string, value: any) => {
        setValues(prev => ({ ...prev, [key]: value }));
    };

    const handleRun = async () => {
        // 校验必填项
        const missing = schema.inputs
            .filter(f => f.required && !values[f.key])
            .map(f => f.label);

        if (missing.length > 0) {
            toast.error(`请填写：${missing.join('、')}`);
            return;
        }

        setStatus('running');
        const prompt = buildPrompt(schema, values);

        try {
            const res = await callAIAgent(schema.id, prompt, schema.advanced);
            setResult(res);
            setStatus('done');
        } catch (e) {
            setStatus('error');
        }
    };

    return (
        <div className="ai-agent-panel">
            <header>
                <span>{schema.icon}</span>
                <h3>{schema.name}</h3>
                <p>{schema.description}</p>
            </header>

            <AgentConfigForm
                schema={schema}
                values={values}
                onChange={handleChange}
            />

            <button
                onClick={handleRun}
                disabled={status === 'running'}
            >
                {status === 'running' ? '生成中...' : '运行'}
            </button>

            {status === 'done' && result && (
                <AgentResultPreview result={result} outputType={schema.output.type} />
            )}
        </div>
    );
}
```

## 15.8 AI 接口超时处理

```javascript
// 1. AbortController 超时控制
async function fetchWithTimeout(url, options = {}, timeout = 30000) {
    const controller = new AbortController();
    const id = setTimeout(() => controller.abort(), timeout);
    
    try {
        const response = await fetch(url, {
            ...options,
            signal: controller.signal
        });
        clearTimeout(id);
        return response;
    } catch (error) {
        clearTimeout(id);
        if (error.name === 'AbortError') {
            throw new Error('请求超时');
        }
        throw error;
    }
}

// 2. 重试机制
async function fetchWithRetry(url, options = {}, maxRetries = 3) {
    for (let i = 0; i < maxRetries; i++) {
        try {
            return await fetch(url, options);
        } catch (error) {
            if (i === maxRetries - 1) throw error;
            await new Promise(r => setTimeout(r, 1000 * (i + 1))); // 指数退避
        }
    }
}

// 3. AI 流式响应超时
async function streamAIResponse(messages, onChunk, options = {}) {
    const { timeout = 60000, retries = 2 } = options;
    
    for (let attempt = 0; attempt <= retries; attempt++) {
        try {
            const controller = new AbortController();
            const timeoutId = setTimeout(() => controller.abort(), timeout);
            
            const response = await fetch('/api/ai/chat', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify({ messages }),
                signal: controller.signal
            });
            
            clearTimeout(timeoutId);
            
            const reader = response.body.getReader();
            const decoder = new TextDecoder();
            
            while (true) {
                const { done, value } = await reader.read();
                if (done) break;
                
                const chunk = decoder.decode(value);
                onChunk(chunk);
            }
            
            return; // 成功
            
        } catch (error) {
            if (attempt === retries) {
                throw new Error(`AI 响应失败，已重试 ${retries} 次`);
            }
            await new Promise(r => setTimeout(r, 1000));
        }
    }
}
```

**传统 XMLHttpRequest（XHR）：**

```javascript

    xhr.open('POST', '/api/ai/chat', true);

    xhr.onprogress = (e) => {
        // XHR 的 progress 事件
```

**Fetch vs XHR 对比：**

| 特性 | Fetch + Stream | XHR |
|------|---------------|-----|
| 流式读取 | ✅ ReadableStream | ⚠️ 需要 onprogress 轮询 |
| 取消请求 | ✅ AbortController | ⚠️ xhr.abort() |
| Promise 化 | ✅ 天然 async/await | ❌ 回调方式 |
| 错误处理 | ⚠️ 只reject网络错误，4xx不算 | ✅ 可区分状态码 |
| 兼容性 | 现代浏览器 | 全部支持 |
| 中止生成 | ✅ 精确控制 | ✅ 可行但不够灵活 |

**题目：在处理 AI 流式输出时，Fetch API 与传统 XMLHttpRequest 的区别是什么？**

**满分答案：**

**Fetch + ReadableStream 的优势：**
1. **真正的流式读取**：`response.body.getReader()` 按块读取，而非轮询
2. **AbortController**：精确控制请求中断，配合 AI "中止生成" 场景
3. **Promise 原生**：可用 async/await，代码更简洁

**XHR 的局限性：**
- `onprogress` 本质是轮询，不是真正的流
- 无法精确控制流式中断

**Fetch 的注意点：**
- 4xx/5xx 不会 reject，需要手动判断 `response.ok`
- `response.body` 只在 Firefox/Chrome 现代浏览器可用

## 15.10 手写 SSE 解析器

#### 知识点详解

**SSE 格式：**

```
event: message
data: {"content": "Hello"}
data: {"content": " world"}

event: done
data: [DONE]
```

**完整解析器实现：**

```javascript
/**
 * SSE 事件解析器
 * 将 SSE 格式字符串解析为事件对象
 */
class SSEParser {
    constructor(onEvent, onError) {
        this.onEvent = onEvent;
        this.onError = onError;
        this.buffer = '';       // 未完成的行缓冲
        this.currentEvent = {}; // 当前事件
    }

    // 处理传入的文本块（来自 Fetch Stream）
    parse(chunk) {
        this.buffer += chunk;
        const lines = this.buffer.split('\n');
        // 最后一行可能是不完整的，保持在 buffer 中
        this.buffer = lines.pop() || '';

        for (const line of lines) {
            this._processLine(line);
        }
    }

    _processLine(line) {
        // 空行：事件结束，触发回调
        if (line === '') {
            if (this.currentEvent.data || this.currentEvent.event) {
                this.onEvent({
                    event: this.currentEvent.event || 'message',
                    data: this.currentEvent.data || '',
                    // 解析 JSON data（如果可以）
                    json: this._tryParseJSON(this.currentEvent.data)
                });
            }
            this.currentEvent = {};
            return;
        }

        // 解析 field: value
        const colonIndex = line.indexOf(':');
        if (colonIndex === -1) return;

        const field = line.slice(0, colonIndex).trim();
        const value = line.slice(colonIndex + 1).trim();

        switch (field) {
            case 'event':
                this.currentEvent.event = value;
                break;
            case 'data':
                // data 字段可能分多行，用换行拼接
                this.currentEvent.data = (this.currentEvent.data || '') + value + '\n';
                break;
            case 'id':
                this.currentEvent.id = value;
                break;
            case 'retry':
                this.currentEvent.retry = parseInt(value, 10);
                break;
        }
    }

    _tryParseJSON(str) {
        try {
            return JSON.parse(str.trim());
        } catch {
            return null;
        }
    }

    // 结束解析
    end() {
        if (this.buffer) {
            this._processLine(this.buffer);
            this.buffer = '';
        }
    }
}

/**
 * Fetch + SSE 的完整使用示例
 */
```

```javascript
function StreamingMarkdownRenderer({ text }) {

        if (!containerRef.current) return;

        const debouncedParse = debounce(() => {

            const safeHtml = DOMPurify.sanitize(html);

            containerRef.current.innerHTML = safeHtml;

            // 4. 渲染数学公式
            if (window.MathJax) {

            }
        }, 300);

        debouncedParse();

        return () => debouncedParse.cancel();
    }, [text]);
```

## 15.14 AI Agent 多步任务状态流转管理

#### 真实面试题（补充）

**题目：复杂的 AI Agent 交互中，如何管理多步任务的状态流转？**

**满分答案：**

**状态机设计：**

```javascript
// AI Agent 任务状态机
const TaskState = {
    IDLE: 'idle',           // 空闲
    RUNNING: 'running',      // 执行中
    WAITING_INPUT: 'waiting', // 等待用户输入
    PAUSED: 'paused',       // 暂停
    COMPLETED: 'completed',  // 完成
    FAILED: 'failed',        // 失败
    CANCELLED: 'cancelled'   // 取消
};

class AITaskManager {
    constructor() {
        this.state = TaskState.IDLE;
        this.steps = [];      // 任务步骤
        this.currentStep = 0;
        this.listeners = new Set();
    }

    // 状态流转
    transition(newState) {
        const oldState = this.state;
        this.state = newState;
        this.notify({ oldState, newState });
    }

    // 订阅状态变化
    subscribe(listener) {
        this.listeners.add(listener);
        return () => this.listeners.delete(listener);
    }

    notify(event) {
        this.listeners.forEach(l => l(event));
    }

    // 执行任务
    async execute(taskDefinition) {
        this.transition(TaskState.RUNNING);
        this.steps = taskDefinition.steps;

        for (let i = 0; i < this.steps.length; i++) {
            this.currentStep = i;
            const step = this.steps[i];

            try {
                const result = await this.executeStep(step);
                step.result = result;
                step.status = 'completed';

                // 检查是否需要用户确认
                if (step.requireUserConfirm) {
                    this.transition(TaskState.WAITING_INPUT);
                    const userInput = await this.waitForUserInput();
                    if (userInput.action === 'cancel') {
                        this.transition(TaskState.CANCELLED);
                        return;
                    }
                    this.transition(TaskState.RUNNING);
                }
            } catch (error) {
                step.error = error;
                step.status = 'failed';
                this.transition(TaskState.FAILED);
                return;
            }
        }

        this.transition(TaskState.COMPLETED);
    }
}
```

```javascript
// 问题：内容非常长时，DOM节点过多导致卡顿
// 解决方案：虚拟滚动 + 分段渲染

class StreamingRenderer {
    constructor(container, options = {}) {
        this.container = container;
        this.buffer = '';           // 文本缓冲区
        this.renderThreshold = 100; // 每100字符渲染一次
        this.maxDomNodes = 1000;    // 最大DOM节点数
        this.virtualScroll = new VirtualScroll(container);
    }

    append(chunk) {
        this.buffer += chunk;
        
        // 累积到一定量再渲染，减少DOM操作
        if (this.buffer.length >= this.renderThreshold) {
            this.flush();
        }
        
        // 使用 requestAnimationFrame 节流
        this.scheduleRender();
    }

    flush() {
        if (this.buffer.length === 0) return;
        
        // 分段渲染，避免一次性插入大量内容
        const segments = this.splitIntoSegments(this.buffer);
        segments.forEach(segment => {
            this.virtualScroll.append(segment);
        });
        
        this.buffer = '';
        
        // 控制DOM节点数量
        if (this.virtualScroll.nodeCount > this.maxDomNodes) {
            this.virtualScroll.recycleOldNodes();
        }
    }

    scheduleRender() {
        if (this.renderScheduled) return;
        this.renderScheduled = true;
        
        requestAnimationFrame(() => {
            this.flush();
            this.renderScheduled = false;
        });
    }
}
```

```javascript
class TokenEstimator {
    // 粗略估算：1 token ≈ 0.75个英文单词 或 1-2个中文字符
    static estimate(text) {
        const chineseChars = (text.match(/[\u4e00-\u9fa5]/g) || []).length;
        const englishWords = text.split(/\s+/).filter(w => /[a-zA-Z]/.test(w)).length;
        return Math.ceil(chineseChars * 1.5 + englishWords * 1.3);
    }

    // 更准确的估算（使用tiktoken的wasm版本）
    static async accurateEstimate(text, model = 'gpt-3.5-turbo') {
        const { encoding_for_model } = await import('tiktoken');
        const encoder = encoding_for_model(model);
        const tokens = encoder.encode(text);
        return tokens.length;
    }

    // 截断历史消息以适应上下文窗口
    static truncateMessages(messages, maxTokens = 4096) {
        let totalTokens = 0;
        const result = [];
        
        // 从最新的消息开始倒序遍历
        for (let i = messages.length - 1; i >= 0; i--) {
            const msg = messages[i];
            const tokens = this.estimate(msg.content);
            
            if (totalTokens + tokens > maxTokens * 0.9) { // 留10%余量
                break;
            }
            
            totalTokens += tokens;
            result.unshift(msg);
        }
        
        return result;
    }
}
```

```javascript
        return results.sort((a, b) => {
            const scoreA = this.calculateRelevance(a, userIntent);
            const scoreB = this.calculateRelevance(b, userIntent);
            return scoreB - scoreA;
        });
    }

    // 4. 渐进式展示（先展示检索结果，再展示AI回复）
    static async streamRAGResponse(query) {
        // 先展示"正在搜索..."
        showSearching();
        
        const docs = await this.retrieveWithCache(query);
        showRetrievedDocs(docs); // 用户可提前看到参考文档
        
        // 再流式展示AI回复
        const stream = await this.generateWithRAG(query, docs);
        return stream;
    }
}
```

```javascript
// 策略1：滑动窗口（保留最近N轮）
class SlidingWindowStrategy {
    constructor(maxRounds = 10) {
        this.maxRounds = maxRounds;
    }
    
    manage(messages) {
        return messages.slice(-this.maxRounds * 2); // 每轮包含user+assistant
    }
}

// 策略2：Token限制（保留最近N个Token）
class TokenLimitStrategy {
    constructor(maxTokens = 4000) {
        this.maxTokens = maxTokens;
    }
    
    manage(messages) {
        let totalTokens = 0;
        const result = [];
        
        for (let i = messages.length - 1; i >= 0; i--) {
            const tokens = estimateTokens(messages[i].content);
            if (totalTokens + tokens > this.maxTokens) break;
            totalTokens += tokens;
            result.unshift(messages[i]);
        }
        
        return result;
    }
}

// 策略3：摘要压缩（早期对话生成摘要）
class SummarizationStrategy {
    async manage(messages, threshold = 10) {
        if (messages.length <= threshold) return messages;
        
        // 早期对话生成摘要
        const oldMessages = messages.slice(0, -threshold);
        const recentMessages = messages.slice(-threshold);
        
        const summary = await this.summarize(oldMessages);
        
        return [
            { role: 'system', content: `历史对话摘要：${summary}` },
            ...recentMessages
        ];
    }
}

// 策略4：分层存储（本地存储+服务端存储）
class HierarchicalStorage {
    // 本地：当前会话（内存）
    // IndexedDB：近期会话
    // 服务端：历史会话
    
    async getContext(sessionId) {
        // 1. 从内存获取当前会话
        const current = this.memory.get(sessionId);
        
        // 2. 从IndexedDB获取近期上下文
        const recent = await this.indexedDB.getRecent(sessionId);
        
        // 3. 从服务端获取历史摘要
        const history = await this.api.getSummary(sessionId);
        
        return {
            current,
            recent,
            history: history ? `历史摘要：${history}` : ''
        };
    }
}
```

**题目1：在处理AI聊天回复时，常使用SSE技术，它和WebSocket有什么区别？**

**满分答案：**

| 维度 | SSE | WebSocket |
|------|-----|-----------|
| 通信方向 | 单向（服务器→客户端） | 双向 |
| 协议 | HTTP | WebSocket |
| 重连 | 浏览器自动重连 | 手动实现 |
| 适用场景 | AI流式回复、股票行情 | 即时通讯、游戏 |

**AI场景选SSE的原因：**
1. AI回复是单向流，无需客户端向服务器实时推送
2. 基于HTTP，穿透防火墙更容易
3. 浏览器自动重连，断线恢复简单
4. 实现成本低，无需处理WebSocket握手和心跳

```javascript
// 方案1：粗略估算（前端实时）
const estimateTokens = (text) => {
    const chinese = (text.match(/[\u4e00-\u9fa5]/g) || []).length;
    const english = text.split(/\s+/).filter(w => /[a-zA-Z]/.test(w)).length;
    return Math.ceil(chinese * 1.5 + english * 1.3);
};

// 方案2：使用tiktoken wasm（更准确）
import { encoding_for_model } from 'tiktoken';
const encoder = encoding_for_model('gpt-3.5-turbo');
const tokens = encoder.encode(text).length;

// 方案3：截断历史消息
const truncateMessages = (messages, maxTokens = 4096) => {
    let total = 0;
    const result = [];
    for (let i = messages.length - 1; i >= 0; i--) {
        const tokens = estimateTokens(messages[i].content);
        if (total + tokens > maxTokens * 0.9) break;
        total += tokens;
        result.unshift(messages[i]);
    }
    return result;
};
```

- 前端实时估算，超过阈值时提示用户

- 预留10%余量（系统提示词也会占用Token）

**题目：解释一下什么是 Prompt Engineering（提示工程）中的 Few-shot 模式？**

**满分答案：**

**核心概念：**
Few-shot = 给 AI 提供少量（通常 1-5 个）示例，让它学习模式后回答新问题。相比 Zero-shot（无示例），Few-shot 大幅提升任务准确率。

**三种模式对比：**

| 模式 | 示例数量 | 适用场景 | 准确率 |
|------|---------|---------|--------|
| Zero-shot | 0 | 简单任务、通用指令 | 中 |
| One-shot | 1 | 格式要求明确的场景 | 较高 |
| Few-shot | 2-5 | 复杂格式、领域特定 | 高 |
| CoT（思维链） | 3-5 | 推理类问题 | 最高 |

**Few-shot 示例结构：**

```
请将以下句子改写成正式商务邮件风格。

示例1：
原句："你好，我想问一下那个项目什么时候能弄完？"
改写："您好，请问该项目的预计完成时间是？"

示例2：
原句："这个方案不行，太烂了"
改写："此方案在可行性方面存在挑战，建议进一步优化。"

示例3：
原句："快点弄，别磨蹭"
改写："请尽快推进，谢谢。"

现在请改写：
原句："用户输入"
改写：
```

**AI 前端实现要点：**

```typescript
// Few-shot 模板管理
class PromptTemplate {
  private shots: Array<{ input: string; output: string }> = [];

  addShot(input: string, output: string): this {
    this.shots.push({ input, output });
    return this;
  }

  build(userInput: string): string {
    const examples = this.shots.map(
      s => '示例：
输入：' + s.input + '
输出：' + s.output
    ).join('

');
    return (
      this.systemPrompt + '

' +
      examples + '

' +
      '现在请回答：
' +
      '输入：' + userInput + '
' +
      '输出：'
    ).trim();
  }

  get tokenCount(): number {
    return estimateTokens(this.build(''));
  }
}
```

- 提到"示例数量受 Token 限制影响"
