## 16.1 一面：技术基础

### 16.1.1 SSE 与 WebSocket 的区别

#### 知识点详解

**SSE（Server-Sent Events）：**

```javascript
// 客户端建立 SSE 连接
const eventSource = new EventSource('/api/stream');
eventSource.onmessage = (event) => {
    console.log('收到数据:', event.data);
};
// 浏览器自动重连
```

**WebSocket：**

```javascript
const ws = new WebSocket('wss://example.com/ws');
ws.onopen = () => ws.send(JSON.stringify({ message: '你好' }));
ws.onmessage = (event) => console.log('收到:', event.data);
// 需要手动实现重连
```

**核心对比：**

| 特性 | SSE | WebSocket |
|------|-----|-----------|
| 通信方向 | 服务器→客户端单向 | 全双工双向 |
| 协议 | HTTP（text/event-stream） | WebSocket（ws/wss） |
| 自动重连 | ✅ 浏览器原生 | ❌ 需手动 |
| 二进制数据 | ❌ 仅文本 | ✅ 支持 |
| 防火墙穿透 | 易 | 可能受限 |

#### 真实面试题

**题目：SSE 和 WebSocket 的区别？AI 聊天为何选 SSE？**

**满分答案：**

**SSE 与 WebSocket 区别：**

1. **通信方向**：SSE 单向（服务器推送），WebSocket 全双工
2. **协议**：SSE 基于 HTTP，WebSocket 独立协议
3. **自动重连**：SSE 原生支持，WebSocket 需手动实现
4. **数据类型**：SSE 仅文本，WebSocket 支持二进制

**AI 聊天选 SSE 的原因：**

1. **单向通信足够**：AI 回复是服务器推送，无需客户端主动发消息
2. **基于 HTTP**：易穿透防火墙，兼容性好
3. **自动重连**：长会话友好
4. **实现简单**：

```javascript
// SSE 实现 AI 流式聊天
const source = new EventSource('/api/chat?message=你好');
source.onmessage = (e) => appendText(e.data);
```

```typescript
function* sentenceGenerator(stream: string): Generator<string> {
    const sentenceEndings = /[.!?。！？]+/g;
```

### 16.1.6 Promise.all vs Promise.allSettled

#### 知识点详解

| 特性 | Promise.all | Promise.allSettled |
|------|-------------|-------------------|
| 成功条件 | 所有成功 | 所有完成 |
| 失败行为 | 立即 reject | 等待全部 |
| 返回值 | 结果数组 | 状态对象数组 |

```typescript
// 多模型对比 → allSettled
const results = await Promise.allSettled([
    callOpenAI(prompt),
    callClaude(prompt),
    callGemini(prompt)
]);
results.forEach((r, i) => {
    if (r.status === 'fulfilled') useResponse(i, r.value);
    else showError(i, r.reason);
});
```

#### 真实面试题

**题目：Promise.all 和 Promise.allSettled 的区别？AI 模型调用选哪个？**

**满分答案：**

**区别：**

| 特性 | Promise.all | Promise.allSettled |
|------|-------------|-------------------|
| 成功条件 | 全部成功 | 全部完成 |
| 失败行为 | 立即 reject | 等待全部完成 |
| 返回值 | 结果数组 | `{status, value/reason}` 数组 |

**AI 模型调用选择：**

**选 Promise.allSettled：**
- 多模型对比展示
- 单个失败不影响其他
- 需要知道每个模型结果

```typescript
const results = await Promise.allSettled([
    callOpenAI(message),
    callClaude(message)
]);
results.forEach((r, i) => {
    if (r.status === 'fulfilled') showModelResponse(i, r.value);
    else showModelError(i);
});
```

**选 Promise.all：**
- 数据聚合分析（需全部成功）
- 任一失败则整体失败

### 16.2.3 Prompt Injection 防御

#### 知识点详解

**攻击类型：**
1. 直接注入：用户输入包含恶意指令
2. 间接注入：通过外部数据注入
3. 越狱攻击：绕过安全限制

**前端防御措施：**

```typescript
// 1. 输入过滤
function sanitizeInput(input: string): string {
    return input
        .replace(/[<>]/g, '')
        .replace(/["'`]/g, '\\$&')
        .trim();
}

// 2. 敏感词检测
const PATTERNS = [
    /ignore previous instructions/i,
    /disregard.*prompt/i,
    /DAN.*mode/i
];
function detectInjection(input: string): boolean {
    return PATTERNS.some(p => p.test(input));
}

// 3. 输入长度限制
const MAX_LENGTH = 4000;

// 4. 上下文隔离
function createContext(segments: Message[]) {
    return segments.map(s => `[${s.role}]: ${s.content}`).join('\n---\n');
}
```

#### 真实面试题

**题目：Prompt Injection 理解？前端能做哪些防御？**

**满分答案：**

**Prompt Injection：** 攻击者通过输入嵌入恶意指令，让 AI 执行非预期操作。

**前端防御措施：**

1. **输入过滤**：移除特殊字符，转义引号
2. **敏感词检测**：检测 "ignore instructions" 等模式
3. **长度限制**：防止超长输入
4. **上下文隔离**：明确标记用户输入边界
5. **后端配合**：后端深度检测，前端初筛

### 16.2.5 AI 对话 Context 管理

#### 知识点详解

**Context 管理策略：**

```typescript
// 1. 本地 Token 估算
function estimateTokens(text: string): number {
    return Math.ceil(text.length / 4);  // 粗略估算
}

// 2. 上下文窗口管理
class ContextManager {
    private messages: Message[] = [];
    private maxTokens: number;
    
    add(message: Message) {
        this.messages.push(message);
        while (this.getTotalTokens() > this.maxTokens) {
            this.messages.shift();  // 移除最早的
        }
    }
    
    getMessages(): Message[] {
        return this.messages;
    }
}

// 3. 滑动窗口策略
function trimContext(messages: Message[], maxTokens: number): Message[] {
    while (estimateTokens(messages) > maxTokens) {
        messages.splice(1, 1);  // 移除中间消息
    }
    return messages;
}
```

#### 真实面试题

**题目：如何管理 AI 对话 Context？前端参与 Token 计算吗？**

**满分答案：**

**Context 管理策略：**

1. **本地 Token 估算**
   - 按字符数粗略估算（中文约 1 字符 ≈ 1 Token）
   - 调用 Tokenizer 库精确计算

2. **滑动窗口**
   - 超出 maxTokens 时移除最早消息
   - 保留系统提示和最近对话

3. **摘要压缩**
   - 对长对话生成摘要替换历史

**前端参与 Token 计算：**

**前端做：**
- 粗略估算（提示用户输入过长）
- 显示 Token 消耗统计

**后端做：**
- 精确计算（模型相关）
- 强制截断（安全边界）

### 16.2.7 前端 RAG 实现

#### 知识点详解

**RAG 流程：**

```
文档 → 解析 → 分块 → 向量化 → 索引 → 检索 → 增强 → 生成
```

**本地文档预处理：**

```typescript
// 1. 文件解析
import { PDFLoader } from 'langchain/document_loaders/fs/pdf';

// 2. 文本分块
import { RecursiveCharacterTextSplitter } from 'langchain/text_splitter';
const splitter = new RecursiveCharacterTextSplitter({
    chunkSize: 500,
    chunkOverlap: 50
});
const chunks = await splitter.splitDocuments(docs);

// 3. 向量化
import { OpenAIEmbeddings } from 'langchain/embeddings/openai';
const embeddings = new OpenAIEmbeddings();
const vectors = await embeddings.embedDocuments(chunks);

// 4. 本地索引
import { LocalIndex } from ' векторная-bd';
const index = new LocalIndex();
index.add(vectors, chunks);

// 5. 相似度检索
const results = await index.search(queryVector, k: 5);
```

#### 真实面试题

**题目：前端如何实现 RAG？本地文档预处理怎么做？**

## 16.1.7 CSS Flex 布局与 flex:1 简写

### 知识点详解

**Flex 布局核心概念：**

Flexbox（Flexible Box）是 CSS3 提出的一种一维布局模型，用于在主轴和交叉轴上灵活分配空间。

```css
.container {
    display: flex;                    /* 启用 Flex 布局 */
    flex-direction: row;              /* 主轴方向：水平（默认） */
    justify-content: space-between;   /* 主轴对齐方式 */
    align-items: center;              /* 交叉轴对齐方式 */
    flex-wrap: wrap;                  /* 换行控制 */
    gap: 16px;                       /* 项目间距 */
}
```

**flex:1 的完整含义：**

`flex:1` 是三个属性的简写（全写为 `flex: 1 1 0%`）：

```css
.item {
    flex-grow: 1;    /* 扩展比例：占据剩余空间的 1 份 */
    flex-shrink: 1;  /* 收缩比例：空间不足时收缩 1 份 */
    flex-basis: 0%;  /* 基础尺寸：初始大小为 0%（内容不占宽） */
}
```

**`flex:1` vs `flex:1 1 auto` vs `flex:1 0 auto`：**

| 属性 | 扩展 | 收缩 | 基础尺寸 | 典型场景 |
|------|------|------|---------|---------|
| `flex: 1` | ✅ | ✅ | `0%` | 等分剩余空间，内容不参与分配 |
| `flex: 1 1 auto` | ✅ | ✅ | `auto` | 基准为自身内容，再分配剩余空间 |
| `flex: 0 0 200px` | ❌ | ❌ | `200px` | 固定尺寸，不伸缩 |

```css
/* 等分布局（常见于导航栏） */
.nav { display: flex; }
.nav-item { flex: 1; text-align: center; }

/* 左右固定，中间自适应（三栏布局） */
.layout { display: flex; }
.sidebar-left  { flex: 0 0 200px; }
.content       { flex: 1; }         /* 中间区域占满剩余 */
.sidebar-right { flex: 0 0 200px; }
```

#### 真实面试题

**题目：简述 CSS 中的 Flex 布局，并说明 `flex:1` 是哪些属性的简写？**

**满分答案：**

**Flex 布局是什么：**
Flexbox 是 CSS3 一维布局模型，通过 `display: flex` 启用。容器内的子元素（Flex 项目）可以在主轴和交叉轴上灵活分配空间、对齐。

**核心属性：**

```css
.container {
    display: flex;
    flex-direction: row;           /* 主轴：水平 → 交叉轴：垂直 */
    justify-content: space-between; /* 主轴对齐 */
    align-items: center;            /* 交叉轴对齐 */
}
```

**`flex: 1` 的完整含义：**

`flex: 1` = `flex-grow: 1; flex-shrink: 1; flex-basis: 0%;`

| 子属性 | 值 | 含义 |
|--------|-----|------|
| `flex-grow` | `1` | 扩展比例 = 1份（参与分配剩余空间） |
| `flex-shrink` | `1` | 收缩比例 = 1份（空间不足时等比收缩） |
| `flex-basis` | `0%` | 基础尺寸 = 0%（内容不占初始宽度） |

**常见误区：**
- `flex: 1` ≠ `flex: 1 1 auto`（后者基础尺寸是内容实际宽度）
- `flex: auto` = `flex: 1 1 auto`（自动尺寸 + 等比伸缩）
- `flex: none` = `flex: 0 0 auto`（完全不伸缩）

**AI 组件应用场景：**
```css
/* 消息气泡靠右/左，flex 处理对齐 */

.avatar       { flex: 0 0 40px; }
.message-body { flex: 1; min-width: 0; }  /* min-width:0 防文字溢出 */
```

## 16.2.9 前端 Token 计算与 Context 限制

### 知识点详解

**Token 估算方法：**

```typescript
// 方法一：粗略估算（快速，不精确）
function estimateTokens(text: string): number {
    // 英文：按空格/词分割，平均每词 ~1.3 token
    // 中文：按字符计，约 1~2 token/字（GPT 模型）
    const chineseChars = (text.match(/[\u4e00-\u9fa5]/g) || []).length;
    const englishWords = text.trim().split(/\s+/).length;
    return Math.ceil(chineseChars * 1.5 + englishWords * 1.3);
}

// 方法二：使用 TikToken 精确计算

const encoder = tiktoken.get_encoding('cl100k_base');

function countTokens(text: string): number {
    const tokens = encoder.encode(text);
    return tokens.length;
}

// 方法三：调用 API 获取精确值（推荐在发送前用）
const response = await openai.chat.completions.create({
    messages: [{ role: 'user', content: prompt }],
    max_tokens: 4096 - estimatedTokens
});
```

**Context 窗口管理策略：**

```typescript
class ContextManager {
    private messages: Message[] = [];
    private maxTokens: number;
    private systemPromptTokens: number;

    constructor(maxTokens: number, systemPrompt: string) {
        this.maxTokens = maxTokens;
        this.systemPromptTokens = estimateTokens(systemPrompt);
    }

    add(message: Message): void {
        this.messages.push(message);
        this.trim();
    }

    private trim(): void {
        // 留 10% 安全边界
        const available = this.maxTokens * 0.9 - this.systemPromptTokens;
        let used = this.messages.reduce((sum, m) => sum + estimateTokens(m.content), 0);

        while (used > available && this.messages.length > 1) {
            const removed = this.messages.shift()!;
            used -= estimateTokens(removed.content);
        }
    }

    getMessages(): Message[] {
        return this.messages;
    }

    getTotalTokens(): number {
        const contentTokens = this.messages.reduce((sum, m) => sum + estimateTokens(m.content), 0);
        return this.systemPromptTokens + contentTokens;
    }
}
```

## 16.2.10 带并发限制的 Promise 调度器

### 知识点详解

**核心思想：**
- 同时最多执行 N 个异步任务
- 新任务排队等待，空闲时自动执行
- 任务完成后自动补充执行

```typescript
class PromiseScheduler {
    private running = 0;
    private queue: Array<() => void> = [];

    constructor(private limit: number) {}

    async add<T>(fn: () => Promise<T>): Promise<T> {
        return new Promise((resolve, reject) => {
            this.queue.push(() => fn().then(resolve).catch(reject));
            this.process();
        });
    }

    private process(): void {
        while (this.running < this.limit && this.queue.length > 0) {
            const task = this.queue.shift()!;
            this.running++;
            task().finally(() => {
                this.running--;
                this.process(); // 继续处理下一个
            });
        }
    }
}

// 使用示例
const scheduler = new PromiseScheduler(3); // 最多3个并发

const results = await Promise.all([
    scheduler.add(() => callAPI('/model-1')),
    scheduler.add(() => callAPI('/model-2')),
    scheduler.add(() => callAPI('/model-3')),
    scheduler.add(() => callAPI('/model-4')), // 排队等待
    scheduler.add(() => callAPI('/model-5')),
]);
```

#### 真实面试题

**题目：手写代码：实现一个带并发限制的 Promise 调度器。**

**满分答案：**

**核心思路：**
1. 维护一个"正在执行"计数器
2. 任务包装成 Promise 加入队列
3. 每完成一个，从队列取下一个继续执行

**完整实现：**

```typescript
class Scheduler {
    private running = 0;
    private queue: Array<() => Promise<unknown>> = [];

    constructor(private limit: number) {}

    add<T>(fn: () => Promise<T>): Promise<T> {
        return new Promise((resolve, reject) => {
            this.queue.push(() => fn().then(resolve).catch(reject));
            this.drain();
        });
    }

    private drain(): void {
        while (this.running < this.limit && this.queue.length > 0) {
            const task = this.queue.shift()!;
            this.running++;
            task().finally(() => {
                this.running--;
                this.drain(); // 递归：继续执行下一个
            });
        }
    }
}

// ============ 测试用例 ============
const scheduler = new Scheduler(2); // 最多2个并发

async function simulateRequest(id: number, ms: number) {
    console.log(`[${id}] 开始`);
    await new Promise(r => setTimeout(r, ms));
    console.log(`[${id}] 完成`);
    return id;
}

(async () => {
    const results = await Promise.all([
        scheduler.add(() => simulateRequest(1, 1000)),
        scheduler.add(() => simulateRequest(2, 800)),
        scheduler.add(() => simulateRequest(3, 600)), // 等待前面完成
        scheduler.add(() => simulateRequest(4, 400)), // 等待前面完成
    ]);
    console.log('全部完成:', results);
})();
/*
执行顺序：
[1] 开始
[2] 开始
[1] 完成 → [3] 开始
[2] 完成 → [4] 开始
[3] 完成
[4] 完成
全部完成: [1, 2, 3, 4]
*/
```

- 多 AI 模型并发调用（如同时查询 GPT/Claude/Gemini）
- 批量发送 AI 请求时控制并发，防止接口限流
- 限制同时进行的 Embedding 请求数量

```typescript
async function parseDocument(file: File): Promise<Document[]> {

function splitText(text: string, options: SplitOptions): string[] {
    const { chunkSize, overlap, separators } = options;
    const chunks: string[] = [];

    for (const sep of separators) {
        if (text.includes(sep)) {
            const parts = text.split(sep);
            for (let i = 0; i < parts.length; i += Math.ceil(chunkSize / 50)) {
                const chunk = parts.slice(i, i + Math.ceil(chunkSize / 50)).join(sep);
                if (chunk.length <= chunkSize * 1.5) chunks.push(chunk);
            }
            break;
        }
    }
    return chunks;
}
```

```typescript
// 3. 背景预热

    {isComplete ? '已生成' : '取消'}
```

## 16.3.11 多轮对话上下文管理策略

### 知识点详解

**多轮对话的挑战：**

```
问题：上下文窗口有限（4K-128K tokens），对话不能无限增长
解决：滑动窗口、摘要压缩、层级管理
```

**存储策略：**

| 层级 | 存储位置 | 容量 | 生命周期 |
|------|---------|------|---------|
| 会话元数据 | URL 参数 | KB 级 | 页面刷新保留 |
| 消息历史 | IndexedDB | MB 级 | 长期持久化 |
| 摘要 | 内存/DB | KB 级 | 会话存活期间 |
| Token 计数 | 内存 | 实时 | 会话存活期间 |

```typescript
// 多层级存储架构
class ConversationManager {
    // 内存层：当前会话
    private messages: Message[] = [];

    // IndexedDB：持久化存储
    private db = new IndexedDBStore('conversations');

    async save(message: Message): Promise<void> {
        this.messages.push(message);
        await this.db.put(message.id, message);
        this.updateTokenCount();
    }

    async load(sessionId: string): Promise<Message[]> {
        const msgs = await this.db.getAll(sessionId);
        this.messages = msgs;
        return msgs;
    }

    // 滑动窗口裁剪
    trimToContextLimit(maxTokens: number): void {
        while (this.getTokenCount() > maxTokens) {
            this.messages.shift(); // 移除最早的
        }
    }
}

// 摘要压缩（高级）
async function compressHistory(messages: Message[]): Promise<string> {
    const summary = await callAI(`
        将以下对话压缩为摘要（保留关键信息和用户意图）：
        ${messages.map(m => `[${m.role}]: ${m.content}`).join('\n')}
    `);
    return summary;
}
```

**传输优化：**

```typescript

    return await api.get(`/sessions/${sessionId}/messages`, {
        params: { limit: 20, before: cursor }

}

// 2. 差量同步（只传新增消息）

    localVersion: 10,    // 本地最新消息版本

    missing: messages.filter(m => m.version > 5)
```

#### 真实面试题

**题目：假如项目需要支持"多轮对话的上下文管理"，前端在存储和传输上有什么策略？**

**满分答案：**

**存储策略（三级架构）：**

| 层级 | 存储方式 | 容量 | 用途 |
|------|---------|------|------|
| 会话列表 | IndexedDB | 1000+ | 长期保存 |
| 消息内容 | IndexedDB | 10MB/会话 | 离线查看 |

```typescript
    trim(messages: Message[], maxTokens: number): Message[] {
        while (this.tokenCount(messages) > maxTokens) {
            messages.shift(); // 移除最早的
        }
        return messages;
    }

```
