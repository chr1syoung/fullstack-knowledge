## 16.1 一面：技术基础

### 16.1.1 SSE 与 WebSocket 的区别

**SSE（Server-Sent Events）：** 服务器向客户端单向推送，基于 HTTP，自动重连。

**WebSocket：** 全双工双向通信，独立协议，需手动实现重连。

| 特性 | SSE | WebSocket |
|------|-----|-----------|
| 通信方向 | 单向 | 双向 |
| 协议 | HTTP | WebSocket |
| 自动重连 | ✅ | ❌ |
| 二进制数据 | ❌ | ✅ |

**AI 聊天选择 SSE 的原因：**
1. 单向通信足够（AI 回复是服务器推送）
2. 基于 HTTP，兼容性好，易穿透防火墙
3. 自动重连机制
4. 实现简单

### 16.1.4 流式字符串按句子切割

```typescript
function* sentenceGenerator(stream: string): Generator<string> {
    const sentenceEndings = /[.!?。！？]+/g;
    let buffer = '';
    
    for (const char of stream) {
        buffer += char;
        if (sentenceEndings.test(char)) {
            yield buffer.trim();
            buffer = '';
        }
    }
    
    if (buffer.trim()) yield buffer.trim();
}
```

### 16.1.6 Promise.all vs Promise.allSettled

| 特性 | Promise.all | Promise.allSettled |
|------|-------------|-------------------|
| 成功条件 | 所有成功 | 所有完成 |
| 失败行为 | 任一失败立即 reject | 等待全部完成 |
| 返回值 | 结果数组 | 状态对象数组 |

**AI 模型调用选择 Promise.allSettled：** 需要知道每个模型的独立结果，单个失败不影响整体。

### 16.3.5 音文字幕同步高亮

**实现方案：**
1. Web Audio API 分析音频时间
2. 字幕文件解析（SRT、VTT）
3. requestAnimationFrame 同步更新
4. 点击跳转：currentTime 设置
5. 高亮样式：CSS transition

```typescript
// 核心逻辑
audio.addEventListener('timeupdate', () => {
    const currentTime = audio.currentTime;
    const activeSubtitle = subtitles.find(
        s => currentTime >= s.start && currentTime <= s.end
    );
    setActiveId(activeSubtitle?.id);
});

// 点击跳转
const handleSubtitleClick = (subtitle: Subtitle) => {
    audio.currentTime = subtitle.start;
};
```
