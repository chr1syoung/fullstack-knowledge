#### 知识点详解

```javascript
const canvas = document.getElementById('canvas');
const ctx = canvas.getContext('2d');

// 基本图形

// 路径
ctx.beginPath();
ctx.moveTo(50, 50);

ctx.fill();
```

**Canvas 实战：绘制折线图：**

```javascript
function drawLineChart(canvas, data, options = {}) {
    const ctx = canvas.getContext('2d');
    const { width, height } = canvas;
    const padding = options.padding || 40;
    
    const maxValue = Math.max(...data.map(d => d.value));
    const minValue = Math.min(...data.map(d => d.value));
    
    const xStep = (width - padding * 2) / (data.length - 1);
    const yScale = (height - padding * 2) / (maxValue - minValue);
    
    // 坐标轴
    ctx.beginPath();
    ctx.moveTo(padding, padding);
    ctx.lineTo(padding, height - padding);
    ctx.lineTo(width - padding, height - padding);
    ctx.strokeStyle = '#333';
    ctx.stroke();
    
    // 折线
    ctx.beginPath();
    data.forEach((point, i) => {
        const x = padding + i * xStep;
        const y = height - padding - (point.value - minValue) * yScale;
        
        if (i === 0) {
            ctx.moveTo(x, y);
        } else {
            ctx.lineTo(x, y);
        }
    });
    ctx.strokeStyle = '#007bff';
    ctx.lineWidth = 2;
    ctx.stroke();
    
    // 数据点
    data.forEach((point, i) => {
        const x = padding + i * xStep;
        const y = height - padding - (point.value - minValue) * yScale;
        
        ctx.beginPath();
        ctx.arc(x, y, 4, 0, Math.PI * 2);
        ctx.fillStyle = '#007bff';
        ctx.fill();
    });
}
```

### 11.2 SVG

```html
<!-- 基本图形 -->
<svg width="200" height="200">
    <!-- 矩形 -->
    <rect x="10" y="10" width="100" height="50" fill="red" stroke="black" stroke-width="2"/>

    <polygon points="100,10 40,198 190,78 10,78 160,198" fill="yellow" stroke="black"/>
    
    <!-- 路径 -->
    <path d="M 10 10 L 100 10 L 100 100 Z" fill="purple"/>
    
    <!-- 文字 -->
    <text x="10" y="50" font-size="20" fill="black">Hello SVG</text>
    
    <!-- 渐变 -->
    <defs>
        <linearGradient id="grad" x1="0%" y1="0%" x2="100%" y2="0%">
            <stop offset="0%" style="stop-color:rgb(255,255,0);stop-opacity:1" />
            <stop offset="100%" style="stop-color:rgb(255,0,0);stop-opacity:1" />
        </linearGradient>
    </defs>
    <rect x="10" y="10" width="180" height="80" fill="url(#grad)"/>
    
    <!-- 动画 -->
    <circle cx="50" cy="50" r="20" fill="red">
        <animate attributeName="cx" from="50" to="150" dur="2s" repeatCount="indefinite"/>
    </circle>
</svg>
```

**SVG vs Canvas：**

| 特性 | SVG | Canvas |
|------|-----|--------|
| 类型 | 矢量图形 | 位图 |
| DOM | 是（可操作） | 否 |
| 事件 | 支持元素级别 | 需要手动计算 |
| 缩放 | 无损 | 有损 |
| 性能 | 元素多时差 | 大量图形时好 |
| 适用场景 | 图标、图表、地图 | 游戏、图像处理 |

#### 真实面试题

**题目：Canvas 和 SVG 有什么区别？各自适用什么场景？**

**满分答案：**

| 对比维度 | Canvas | SVG |
|---------|--------|-----|
| 渲染方式 | 像素（位图） | 矢量（DOM） |
| 缩放 | 失真 | 不失真 |
| 事件处理 | 需手动计算坐标 | 原生支持 DOM 事件 |
| 性能 | 大量元素时更好 | 元素多时 DOM 操作慢 |
| 动画 | requestAnimationFrame | CSS/SMIL/JS 动画 |
| 适用场景 | 游戏、图像处理、数据可视化 | 图标、地图、少量交互图形 |

**选择原则：**
- 元素数量 > 1000 → Canvas（DOM 操作太慢）
- 需要缩放不失真 → SVG
- 需要复杂交互（hover、click 单个元素）→ SVG
- 视频帧处理、像素操作 → Canvas
