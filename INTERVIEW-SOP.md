## 三、内容写入格式模板

每道题必须包含以下四个部分，缺一不可：

```markdown
## X.X 知识点标题

#### 知识点详解

**核心概念：**
[一句话定义]

**原理/机制：**
[详细讲解，配图或流程说明]

```javascript
// 代码示例（必须可运行）
```

**对比/场景（可选）：**
| 维度 | 方案A | 方案B |
|------|-------|-------|

#### 真实面试题

**题目：[原题，保持原始表述]**

**满分答案：**

[结构化答案，要点分条，有代码示例]

---
```

### 格式检查清单

- [ ] 章节号连续，无跳号
- [ ] 有 `#### 知识点详解` 标题
- [ ] 有代码示例（至少一个）
- [ ] 有 `#### 真实面试题` 标题
- [ ] 有 `**满分答案：**` 标签
- [ ] 末尾有 `---` 分隔符
- [ ] 无重复内容（写入前搜索关键词）

## 四、处理流程（AI 执行步骤）

```
输入：一段面经文本
         ↓
Step 1: 解析题目
  - 逐题提取，编号
  - 识别题目类型（概念/手写/场景/软技能）
         ↓
Step 2: 分类映射
  - 对照「文件分工速查表」确定目标文件
  - 检查目标文件是否已有相关内容（grep 关键词）
  - 已有内容：补充面试题格式到现有章节
  - 无内容：新建章节，分配编号
         ↓
Step 3: 内容写入
  - 按「内容写入格式模板」撰写
  - 知识点详解 + 代码示例 + 面试题 + 满分答案
  - 追加到目标文件末尾（或插入到相关章节后）
         ↓
Step 4: 更新索引
  - 在 interview-materials/补充-YYYY-MM-DD.md 添加索引行
  - 格式：| 题目 | 目标文件 | 章节号 |
         ↓
Step 5: 更新章节号表
  - 更新本文档「当前最大章节号」表
         ↓
Step 6: Git 提交
  - git add -A
  - git commit -m "feat: 补充第X批N道面经\n\n- 文件1: 章节描述\n- 文件2: 章节描述"
  - git push
```

## 五、题目类型处理规范

### 5.1 概念理解题
> 例："谈谈你对闭包的理解"

```
处理方式：
1. 检查目标文件是否已有该知识点
2. 已有 → 在现有章节末尾补充「真实面试题」部分
3. 无 → 新建完整章节（知识点详解 + 面试题）
```

### 5.2 手写代码题
> 例："手写 Promise.all"、"实现深拷贝"

```
处理方式：
1. 归入 02-JavaScript基础与进阶.md（通用工具函数）
2. 知识点详解部分：完整实现代码 + 关键点注释
3. 面试题部分：题目 + 满分答案（含边界条件说明）
4. 代码必须可运行，包含测试用例
```

### 5.3 场景设计题
> 例："如何设计前端监控系统"、"大文件上传方案"

```
处理方式：
1. 归入对应领域文件（性能/安全/新技术等）
2. 知识点详解：架构图（ASCII）+ 核心模块代码
3. 面试题：结构化答案（分点，有数字/指标）
```

### 5.4 软技能题
> 例："职业规划"、"最难的技术挑战"

```
处理方式：
1. 归入 13-软技能.md
2. 知识点详解：回答框架（STAR法则等）+ 示例结构
3. 面试题：满分答案框架 + 关键要点
```

### 5.5 已有内容的题目
> 题目对应的知识点已在知识库中

```
处理方式：
1. 找到现有章节
2. 在章节末尾（---之前）追加：
   #### 真实面试题（补充）
   **题目：[新题目]**
   **满分答案：**
   [答案]
3. 不重复已有内容，只补充差异点
```

## 八、质量标准

### 知识点详解质量标准
- ✅ 有原理解释（不只是用法）
- ✅ 有可运行代码示例
- ✅ 有对比表格或流程图（复杂概念）
- ✅ 有常见误区或注意事项
- ❌ 不能只有代码没有解释
- ❌ 不能只有概念没有代码

### 满分答案质量标准
- ✅ 结构化（分点，有层次）
- ✅ 有数字/指标（"提升60%"而非"提升很多"）
- ✅ 有代码示例（手写题必须）
- ✅ 有延伸（加分项）
- ❌ 不能只有一两句话
- ❌ 不能照抄知识点详解

## 九、常见问题

**Q：题目跨多个领域怎么处理？**
A：选最相关的一个文件，在索引中注明。例如"React性能优化"→ 05-React，不要拆分到09-性能优化。

**Q：题目已有内容，但现有内容太浅怎么处理？**
A：在现有章节内补充深度内容，不新建章节。在章节末尾追加"#### 深度补充"。

**Q：手写题的代码要多完整？**
A：必须包含：完整实现 + 边界条件处理 + 测试用例。不能只写核心逻辑。

**Q：软技能题没有"正确答案"怎么写？**
A：写"回答框架"而非"标准答案"。给出结构模板 + 示例填充。
