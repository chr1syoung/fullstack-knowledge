## 知识点总览

本次补充覆盖以下 CI/CD 核心知识点：

### 基础概念
- **持续集成（CI）**：每次提交触发构建+测试，尽早发现错误
- **持续交付（CD）**：代码始终可部署，人工审批控制发布
- **持续部署（CD）**：全自动化，代码通过测试直接进入生产

### Pipeline 设计
- 完整 13 阶段：触发 → Checkout → Lint → 依赖 → 单元测试 → 集成测试 → 安全扫描 → 构建 → 镜像 → 部署测试 → E2E → 预生产 → 生产
- DORA 四项核心指标：部署频率、前置时间、MTTR、变更失败率

### 部署策略
- **蓝绿部署**：双环境并行，全量瞬间切换，200% 资源
- **金丝雀部署**：渐进流量分配 5%→20%→50%→100%，资源省 10-30%
- **滚动发布**：K8s 默认，逐个替换

### GitOps
- 核心：Git 为唯一真相来源，ArgoCD 自动同步
- 优势：权限更小、配置漂移自动修复、天然审计

### 工具对比
- Jenkins：最灵活但维护成本高
- GitHub Actions：GitHub 原生，生态最丰富
- GitLab CI：GitLab 一体化，配置简洁

### 安全实践
- 十大安全实践（Secrets、最小权限、依赖扫描、镜像签名等）
- 依赖漏洞扫描（Trivy/Snyk）
- Pipeline 安全：GPG 签名、环境隔离

### 回滚机制
- kubectl rollout undo / ArgoCD rollback / Git revert
- 灰度回滚（金丝雀）和全量回滚的决策树
- MTTR 目标：< 6 分钟全链路恢复
