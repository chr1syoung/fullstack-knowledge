## 14.5 WebGPU：AI 视频前端的下一个变革

#### 知识点详解

**WebGPU 是什么？**

WebGPU 是新一代 Web 图形 API，是 WebGL 的继任者，提供对 GPU 的更底层访问能力。

```
WebGL（旧）：基于 OpenGL ES，API 设计老旧，CPU 开销大
WebGPU（新）：基于 Vulkan/Metal/D3D12，现代化设计，支持计算着色器
```

**WebGPU vs WebGL：**

| 特性 | WebGL | WebGPU |
|------|-------|--------|
| 底层 API | OpenGL ES | Vulkan/Metal/D3D12 |
| 计算着色器 | ❌ | ✅ |
| GPU 计算（非图形） | 有限 | 完整支持 |
| 多线程 | 有限 | 支持 |
| 性能 | 较好 | 更好 |
| 浏览器支持 | 全面 | Chrome 113+（逐步普及） |

**WebGPU 基础用法：**

```javascript
// 1. 初始化
const adapter = await navigator.gpu.requestAdapter();
const device = await adapter.requestDevice();

// 2. 创建 Canvas 上下文
const canvas = document.getElementById('canvas');
const context = canvas.getContext('webgpu');
const format = navigator.gpu.getPreferredCanvasFormat();
context.configure({ device, format });

// 3. 计算着色器（WGSL 语言）
const shaderCode = `
@group(0) @binding(0) var<storage, read> input: array<f32>;
@group(0) @binding(1) var<storage, read_write> output: array<f32>;

@compute @workgroup_size(64)
fn main(@builtin(global_invocation_id) id: vec3<u32>) {
    let i = id.x;
    // 图像处理：灰度转换
    output[i] = input[i * 4] * 0.299 +
                input[i * 4 + 1] * 0.587 +
                input[i * 4 + 2] * 0.114;
}
`;

// 4. 创建计算管线
const shaderModule = device.createShaderModule({ code: shaderCode });
const pipeline = device.createComputePipeline({
    layout: 'auto',
    compute: { module: shaderModule, entryPoint: 'main' }
});

// 5. 执行 GPU 计算
async function processImageOnGPU(imageData) {
    // 创建 GPU Buffer
    const inputBuffer = device.createBuffer({
        size: imageData.data.byteLength,
        usage: GPUBufferUsage.STORAGE | GPUBufferUsage.COPY_DST
    });

    const outputBuffer = device.createBuffer({
        size: imageData.data.byteLength / 4,
        usage: GPUBufferUsage.STORAGE | GPUBufferUsage.COPY_SRC
    });

    // 上传数据到 GPU
    device.queue.writeBuffer(inputBuffer, 0, imageData.data);

    // 绑定组
    const bindGroup = device.createBindGroup({
        layout: pipeline.getBindGroupLayout(0),
        entries: [
            { binding: 0, resource: { buffer: inputBuffer } },
            { binding: 1, resource: { buffer: outputBuffer } }
        ]
    });

    // 提交计算命令
    const encoder = device.createCommandEncoder();
    const pass = encoder.beginComputePass();
    pass.setPipeline(pipeline);
    pass.setBindGroup(0, bindGroup);
    pass.dispatchWorkgroups(Math.ceil(imageData.width * imageData.height / 64));
    pass.end();

    device.queue.submit([encoder.finish()]);

    // 读取结果
    const readBuffer = device.createBuffer({
        size: outputBuffer.size,
        usage: GPUBufferUsage.COPY_DST | GPUBufferUsage.MAP_READ
    });

    const readEncoder = device.createCommandEncoder();
    readEncoder.copyBufferToBuffer(outputBuffer, 0, readBuffer, 0, outputBuffer.size);
    device.queue.submit([readEncoder.finish()]);

    await readBuffer.mapAsync(GPUMapMode.READ);
    return new Float32Array(readBuffer.getMappedRange());
}
```

**WebGPU 对 AI 视频前端的改变：**

```
1. 本地推理加速
   - 在浏览器中直接运行小型 AI 模型（如 Stable Diffusion Tiny）
   - WebGPU + ONNX Runtime Web = 本地 AI 推理
   - 无需服务器，保护隐私

2. 实时视频特效
   - GPU 并行处理每一帧
   - 实时风格迁移、背景替换
   - 比 Canvas 2D 快 10-100 倍

3. 视频编解码加速
   - 配合 WebCodecs，GPU 加速编解码
   - 4K 视频实时处理成为可能

4. 大规模数据处理
   - 视频帧的批量 AI 分析
   - 实时目标检测、场景识别
```

**错误采集：**

```javascript
class ErrorMonitor {
    init() {
        // 1. JS 运行时错误
        window.addEventListener('error', (event) => {
            // 区分 JS 错误和资源加载错误
            if (event.target && (event.target.src || event.target.href)) {
                // 资源加载失败
                this.report({
                    type: 'resource_error',
                    url: event.target.src || event.target.href,
                    tagName: event.target.tagName
                });
            } else {
                // JS 运行时错误
                this.report({
                    type: 'js_error',
                    message: event.message,
                    filename: event.filename,
                    lineno: event.lineno,
                    colno: event.colno,
                    stack: event.error?.stack
                });
            }
        }, true);  // 捕获阶段，能捕获资源错误

        // 2. Promise 未捕获异常
        window.addEventListener('unhandledrejection', (event) => {
            this.report({
                type: 'promise_error',
                message: event.reason?.message || String(event.reason),
                stack: event.reason?.stack
            });
        });

        // 3. React 错误边界（需在组件中配合使用）
        // 4. Vue errorHandler
        // app.config.errorHandler = (err) => this.report(...)
    }

    report(data) {
        const payload = {
            ...data,
            url: location.href,
            userAgent: navigator.userAgent,
            timestamp: Date.now(),
            userId: this.getUserId(),
            sessionId: this.getSessionId()
        };

        // 使用 sendBeacon 保证页面关闭时也能上报
        navigator.sendBeacon('/api/monitor/error', JSON.stringify(payload));
    }
}
```

**性能采集：**

```javascript
class PerformanceMonitor {
    collectWebVitals() {
        // LCP（最大内容绘制）
        new PerformanceObserver((list) => {
            const entries = list.getEntries();
            const lcp = entries[entries.length - 1];
            this.report({ type: 'LCP', value: lcp.startTime });
        }).observe({ entryTypes: ['largest-contentful-paint'] });

        // FID（首次输入延迟）
        new PerformanceObserver((list) => {
            list.getEntries().forEach(entry => {
                this.report({
                    type: 'FID',
                    value: entry.processingStart - entry.startTime
                });
            });
        }).observe({ entryTypes: ['first-input'] });

        // CLS（累积布局偏移）
        let clsValue = 0;
        new PerformanceObserver((list) => {
            list.getEntries().forEach(entry => {
                if (!entry.hadRecentInput) {
                    clsValue += entry.value;
                }
            });
            this.report({ type: 'CLS', value: clsValue });
        }).observe({ entryTypes: ['layout-shift'] });
    }

    collectPageLoad() {
        window.addEventListener('load', () => {
            const timing = performance.timing;
            this.report({
                type: 'page_load',
                dns: timing.domainLookupEnd - timing.domainLookupStart,
                tcp: timing.connectEnd - timing.connectStart,
                ttfb: timing.responseStart - timing.requestStart,
                domReady: timing.domContentLoadedEventEnd - timing.navigationStart,
                load: timing.loadEventEnd - timing.navigationStart
            });
        });
    }
}
```

**数据上报策略：**

```javascript
class Reporter {
    constructor() {
        this.queue = [];
        this.maxQueueSize = 20;
        this.flushInterval = 5000;  // 5秒批量上报

        // 定时批量上报
        setInterval(() => this.flush(), this.flushInterval);

        // 页面关闭前上报
        window.addEventListener('beforeunload', () => this.flush());
        document.addEventListener('visibilitychange', () => {
            if (document.visibilityState === 'hidden') this.flush();
        });
    }

    add(data) {
        this.queue.push(data);
        // 错误立即上报
        if (data.type?.includes('error')) {
            this.flush();
        } else if (this.queue.length >= this.maxQueueSize) {
            this.flush();
        }
    }

    flush() {
        if (this.queue.length === 0) return;
        const data = [...this.queue];
        this.queue = [];

        // sendBeacon 不受页面关闭影响
        navigator.sendBeacon('/api/monitor/batch', JSON.stringify(data));
    }
}
```

## 14.7 微前端架构

#### 知识点详解

**微前端解决的问题：**

```
大型单体前端应用的痛点：
- 代码库庞大，构建慢
- 多团队协作冲突
- 技术栈无法升级（历史包袱）
- 部署耦合（改一处要全量发布）

微前端解决方案：
- 将大应用拆分为多个独立子应用
- 各子应用独立开发、独立部署
- 主应用负责路由和子应用加载
```

**主流实现方案：**

**方案一：qiankun（阿里，最流行）**

```javascript
// 主应用

registerMicroApps([
    {
        name: 'react-app',
        entry: '//localhost:3001',
        container: '#subapp-container',
        activeRule: '/react',
        props: { token: 'xxx' }  // 传递给子应用的数据
    },
    {
        name: 'vue-app',
        entry: '//localhost:3002',
        container: '#subapp-container',
        activeRule: '/vue'
    }
]);

start({
    sandbox: { strictStyleIsolation: true }  // 样式隔离
});

// 子应用（React）

export function mount(props) {
    ReactDOM.render(<App />, props.container.querySelector('#root'));
}
export function unmount(props) {
    ReactDOM.unmountComponentAtNode(props.container.querySelector('#root'));
}
```

**方案二：Module Federation（Webpack 5）**

**方案三：iframe（最简单，隔离最彻底）**

**方案对比：**

| 方案 | 隔离性 | 通信 | 性能 | 适用场景 |
|------|--------|------|------|---------|
| qiankun | JS沙箱+CSS隔离 | props/事件总线 | 好 | 主流选择 |
| Module Federation | 无隔离 | 直接共享 | 最好 | 同技术栈 |
| iframe | 最彻底 | postMessage | 差 | 强隔离需求 |

## 14.8 技术规范推动与组件库建设

#### 知识点详解

**推动技术规范的步骤：**

```
1. 发现问题 → 量化痛点

   - 记录 Bug 类型分布

   - 写文档、做分享
   - 提供迁移工具
   - 渐进式而非强制

   - 定期 Review
```

## 14.9 老旧项目技术重构与迁移

#### 知识点详解

**重构前的评估：**

```
评估维度：
1. 技术债务量化
   - 代码复杂度（圈复杂度）
   - 测试覆盖率
   - 依赖过期程度

2. 业务影响评估
   - 重构期间的功能稳定性
   - 用户影响范围

3. 收益预估
   - 开发效率提升
   - 性能改善
   - 维护成本降低
```

**渐进式重构策略（绞杀者模式）：**

```javascript
// 绞杀者模式（Strangler Fig Pattern）
// 新功能用新技术栈，老功能逐步迁移，不一次性重写

// 例：jQuery 项目迁移到 React
// 1. 新页面用 React 开发
// 2. 老页面中的独立模块逐步替换
// 3. 最终完全迁移

// 技术手段：在 jQuery 页面中挂载 React 组件
function mountReactComponent(containerId, Component, props) {
    const container = document.getElementById(containerId);
    if (container) {
        ReactDOM.render(<Component {...props} />, container);
    }
}

// 老页面中使用
mountReactComponent('new-feature-container', NewFeatureComponent, {
    data: window.legacyData
});
```

**依赖升级策略：**

```bash
# 1. 检查过期依赖
npm outdated

# 2. 自动升级（谨慎）
npx npm-check-updates -u

# 3. 分批升级（降低风险）
# 先升级开发依赖（ESLint、Prettier）
# 再升级工具链（Webpack、Babel）
# 最后升级核心框架（React、Vue）

# 4. 每次升级后运行测试
npm test
```

**重构检查清单：**

```
代码层面：
□ 消除重复代码（DRY 原则）
□ 拆分大文件（单文件 < 300 行）
□ 统一命名规范
□ 添加 TypeScript 类型

工程层面：
□ 升级构建工具（Webpack → Vite）
□ 添加单元测试
□ 配置 ESLint + Prettier
□ 优化 CI/CD 流程

性能层面：
□ 分析打包体积（Bundle Analyzer）
□ 代码分割（动态 import）
□ 图片优化
□ 缓存策略
```

**风险控制：**

```javascript
// 1. 功能开关（Feature Flag）
// 新旧代码并存，通过开关控制
const useNewFeature = featureFlags.get('new-checkout-flow');

if (useNewFeature) {
    return <NewCheckoutFlow />;
} else {
    return <OldCheckoutFlow />;
}

// 2. 灰度发布
// 先对 5% 用户开放，观察指标，再逐步放量

// 3. 回滚方案
// 保留旧版本代码，出问题立即切回
```

#### 真实面试题

**题目：面对一个老旧且复杂的项目，你会如何进行技术重构或迁移？**

**满分答案：**

**核心原则：渐进式，不一次性重写**

**四步走：**
1. **评估**：量化技术债务，评估业务影响，确定优先级
2. **策略**：绞杀者模式（新功能用新技术，老功能逐步替换）
3. **执行**：分批升级依赖 → 工程化改造 → 核心模块重构
4. **保障**：功能开关 + 灰度发布 + 完善的回滚方案

**关键经验：**
- 先加测试，再重构（没有测试的重构是盲目的）
- 每次只改一件事，保持可回滚
- 用数据验证重构效果（性能指标、错误率）

## 14.10 WebAssembly 在前端领域的发展

### 知识点详解

**WebAssembly（Wasm）的定位：**

```
JavaScript：高级语言，灵活但性能有限
WebAssembly：汇编级性能，接近原生
两者结合：JS 负责逻辑，Wasm 负责计算密集型任务
```

**核心价值：**

| 场景 | JS 性能 | Wasm 性能 | 提升 |
|------|---------|---------|------|
| 图像处理 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 10-100x |
| 音视频编解码 | ⭐ | ⭐⭐⭐⭐⭐ | 50x+ |
| 加密解密 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 5-20x |
| AI 推理 | ⭐⭐ | ⭐⭐⭐⭐ | 5-10x |

**典型应用场景：**

```typescript
// 1. ONNX Runtime Web（Wasm）- AI 推理
import { InferenceSession } from 'onnxruntime-web';

const session = await InferenceSession.create('./model.onnx');
const results = await session.run({ input: tensor });

// 2. FFmpeg.wasm - 视频转码
import { createFFmpeg } from '@ffmpeg/ffmpeg';
const ffmpeg = createFFmpeg({ log: true });
await ffmpeg.load();
ffmpeg.FS('writeFile', 'input.mp4', data);
await ffmpeg.run('-i', 'input.mp4', 'output.webm');

// 3. Sqorn（Wasm 数据库）- 浏览器内 SQL
import { init, query } from 'sqorn-wasm';
await init();
const result = await query`SELECT * FROM users WHERE age > 18`;
```

### 真实面试题

**题目：你如何看待 WebAssembly 在未来前端领域的发展？**

**满分答案：**

**当前应用场景（已经成熟的）：**

| 领域 | 应用 | 代表项目 |
|------|------|---------|
| AI 推理 | 浏览器端模型推理 | ONNX Runtime Web、Transformers.js |
| 音视频 | 浏览器端编解码 | FFmpeg.wasm、WebCodecs |
| 游戏 | 3D 游戏移植 | Unity WebGL、Unreal Wasm |
| CAD/建模 | 专业软件 Web 化 | Figma（部分模块）、TinkerCAD |

**未来发展趋势：**

**1. AI 前端（最火热）**
```typescript
// Transformers.js：浏览器内运行 LLM
import { pipeline } from '@huggingface/transformers';

const classifier = await pipeline('sentiment-analysis');
const result = await classifier('I love this product!');
// 不需要后端，模型在浏览器内推理
```

**2. 性能关键模块 Wasm 化**
- 图片处理：Canvas → Wasm 滤镜
- 数据可视化：大数据量计算用 Wasm
- 加密货币：钱包核心逻辑用 Wasm

**3. 组件化 Wasm**
```typescript
// 未来：npm install @wasm/textract
import { extract } from '@wasm/textract';
// 使用方式和 npm 包完全一样，底层是 Wasm
```

**挑战与局限性：**

| 问题 | 说明 | 现状 |
|------|------|------|
| 启动时间 | .wasm 文件需要加载+编译 | 有 WASM CDN 优化 |
| 调试困难 | 源码映射不完善 | DevTools 在改善 |
| JS/Wasm 互调 | 数据传递有开销 | 接口设计是关键 |
| 内存管理 | Wasm 内存手动管理 | 复杂度提升 |

**前端工程师的应对策略：**

1. **不需要学 Rust/C++**：已有大量成熟 Wasm 库，npm 一键安装
2. **聚焦应用场景**：判断什么时候用 Wasm（计算密集型）
3. **混合架构**：JS 负责 UI/交互，Wasm 负责计算
4. **关注 WebGPU**：Wasm + WebGPU = 浏览器内 AI 推理的黄金组合

**结论：** WebAssembly 不是要替代 JavaScript，而是补足 JS 的性能短板。未来前端工程师不需要写 Wasm，但必须懂得在什么时候用它。
