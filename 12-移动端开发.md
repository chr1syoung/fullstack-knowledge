## 12.x JSBridge 实现原理

#### 知识点详解

**什么是 JSBridge？**

JSBridge 是 H5 与 Native（iOS/Android）之间通信的桥梁，让 JS 能调用原生能力（相机、定位、支付等），Native 也能调用 JS 方法。

```
H5（WebView）  ←→  JSBridge  ←→  Native App
```

**JS 调用 Native 的两种方式：**

**方式一：URL Scheme 拦截**

```javascript
// H5 端：构造特殊 URL
function callNative(method, params, callback) {
    const callbackId = `cb_${Date.now()}`;

    // 注册回调
    window._jsbridge_callbacks = window._jsbridge_callbacks || {};
    window._jsbridge_callbacks[callbackId] = callback;

    // 构造 URL Scheme
    const url = `myapp://jsbridge?method=${method}&params=${encodeURIComponent(JSON.stringify(params))}&callbackId=${callbackId}`;

    // 触发 URL（Native 拦截）
    const iframe = document.createElement('iframe');
    iframe.style.display = 'none';
    iframe.src = url;
    document.body.appendChild(iframe);
    setTimeout(() => document.body.removeChild(iframe), 100);
}

// Native 回调 JS
function _nativeCallback(callbackId, result) {
    const cb = window._jsbridge_callbacks[callbackId];
    if (cb) {
        cb(result);
        delete window._jsbridge_callbacks[callbackId];
    }
}

// 使用
callNative('getLocation', {}, (result) => {
    console.log('位置:', result.latitude, result.longitude);
});
```

**方式二：注入 JS 对象（更现代）**

```javascript
// Android：WebView.addJavascriptInterface()
// iOS：WKScriptMessageHandler

// Native 注入后，H5 可以直接调用
window.NativeBridge.getLocation(JSON.stringify({
    success: 'window._locationCallback'
}));

// 或者使用 iOS 的 webkit.messageHandlers
window.webkit.messageHandlers.getLocation.postMessage({
    callbackId: 'cb_001'
});
```

**Native 调用 JS：**

```javascript
// Native 端执行 JS 代码
// Android: webView.evaluateJavascript("javascript:方法名(参数)", null)
// iOS: webView.evaluateJavaScript("方法名(参数)")

// H5 端暴露方法供 Native 调用
window.onNativeMessage = function(data) {
    const { type, payload } = JSON.parse(data);
    switch (type) {
        case 'userInfo':
            updateUserInfo(payload);
            break;
        case 'networkChange':
            handleNetworkChange(payload);
            break;
    }
};
```

**完整 JSBridge SDK 封装：**

```javascript
class JSBridge {
    constructor() {
        this.callbacks = new Map();
        this.callbackId = 0;

        // 监听 Native 消息
        window.addEventListener('message', this._handleNativeMessage.bind(this));
    }

    // JS 调用 Native
    call(method, params = {}) {
        return new Promise((resolve, reject) => {
            const id = `cb_${++this.callbackId}`;
            this.callbacks.set(id, { resolve, reject });

            const message = JSON.stringify({ method, params, callbackId: id });

            if (window.webkit?.messageHandlers?.bridge) {
                // iOS
                window.webkit.messageHandlers.bridge.postMessage(message);
            } else if (window.AndroidBridge) {
                // Android
                window.AndroidBridge.call(message);
            } else {
                // 降级：模拟或报错
                reject(new Error('非 App 环境'));
            }

            // 超时处理
            setTimeout(() => {
                if (this.callbacks.has(id)) {
                    this.callbacks.delete(id);
                    reject(new Error(`${method} 调用超时`));
                }
            }, 10000);
        });
    }

    // 处理 Native 回调
    _handleNativeMessage(event) {
        const { callbackId, result, error } = JSON.parse(event.data);
        const cb = this.callbacks.get(callbackId);
        if (!cb) return;

        this.callbacks.delete(callbackId);
        error ? cb.reject(new Error(error)) : cb.resolve(result);
    }
}

// 使用
const bridge = new JSBridge();

async function getLocation() {
    try {
        const location = await bridge.call('getLocation');
        console.log(location);
    } catch (e) {
        console.error('获取位置失败:', e.message);
    }
}
```

#### 真实面试题

**题目：在 Hybrid 开发中，JSBridge 的实现原理是什么？**

**满分答案：**

**JS 调用 Native 两种方式：**
1. **URL Scheme 拦截**：H5 构造特殊 URL，Native WebView 拦截并处理
2. **注入 JS 对象**：Native 向 WebView 注入对象（Android `addJavascriptInterface`，iOS `WKScriptMessageHandler`）

**Native 调用 JS：**
- 直接执行 JS 代码：`evaluateJavaScript("window.onMessage(data)")`

**完整通信流程：**
1. H5 调用 → 生成 callbackId → 发消息给 Native
2. Native 处理 → 执行 `evaluateJavaScript` 回调 H5
3. H5 根据 callbackId 找到对应 Promise，resolve/reject
